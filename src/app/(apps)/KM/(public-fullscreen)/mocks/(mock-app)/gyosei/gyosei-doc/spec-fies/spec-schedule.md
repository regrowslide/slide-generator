# 設計書 — スケジュール秘書（AI秘書）

> 最終更新: 2026-03-10（0310-MTG 決定事項を反映）

---

## 概要

顧客との日程調整業務をAIが代行するツール。
カレンダーの空き確認 → 仮押さえ → 日程打診メール送信 → 返信検知 → 確定 のワークフローを自動化する。

- **提供元**: 保利国際法務事務所
- **開発**: 合同会社改善マニア
- **対象ユーザー**: 保利国際法務事務所スタッフ（ログイン必須）

---

## 背景・課題

- TimeRex等の日程調整URLをそのまま送ると「機械的」「失礼だ」と受け取られる顧客がいる
- テキストで候補日を丁寧に打診したいが、手作業が多い（空き確認、仮押さえ、返信待ち、確定登録）
- ダブルブッキングリスク、返信なし放置リスクがある
- 毎月の維持費用をなるべくかからない方法を希望

## ワークフロー

```
① 候補日抽出 → ② 仮押さえ → ③ 日程打診メール → ④ 返信検知・確定 → ⑤ 未返信アラート → ⑥ 再調整
```

### ① 候補日の抽出

- ユーザーがAIに「○○様との打ち合わせ候補日を教えて」と指示
- AI が Googleカレンダーの空き状況を読み取り、候補日時をテキストで提示
- 候補日は複数提示（3〜5候補を目安）

### ② カレンダーの仮押さえ

- ユーザーが「その候補日で仮押さえして」と指示
- AI が Googleカレンダーに仮予定を自動登録
- 仮予定には「仮押さえ」であることが分かるマーク・タグを付与

### ③ 日程打診メールの送信

- 仮押さえした候補日をもとに、AI が日程打診メールの文面を自動作成
- **下書きメールはテキストで画面に表示（即時送信ではない）**
- ユーザーが内容を確認・修正後、承認ボタンで送信
- **人間承認を必ず挟む**（誤送信防止）
- 仮押さえ記録・メール文など全て実行記録として残す

### ④ 返信検知 → 確定

- 顧客からGmail宛に返信があった場合、AIが自動検知
- 確定した日時を正式なスケジュールとして登録
- 他の仮押さえを自動解除
- 確定通知をユーザーに送信

### ⑤ 未返信アラート

- 翌営業日までに返信がない場合、AIからユーザーへ通知
- 「○○様からお返事がありません」とChatWorkへアラート送信
- 営業日判定が必要（土日祝を除外）

### ⑥ 再調整メール

- ユーザーが「仮押さえを解除して」と指示
- AI がカレンダーの仮押さえをすべて削除
- 「ご都合の良い日を教えてください」といった再調整メールを自動作成
- 再度、人間承認を挟んでから送信

| -------------- | --------------------------------------------- |
| Google認証     | 専用アプリ側でOAuth認証（ChatWork経由は不可） |
| カレンダー操作 | Google Calendar API                           |

| 通知           | ChatWork API（通知のみ）                      |
| AI対話         | アプリ内チャットUI                            |
| 返信検知       | Gmail API ポーリング or Pub/Sub               |
| DB             | PostgreSQL（Prisma）                          |

## 設計方針

### ChatWorkは通知専用

- ChatWork上での認証やAPI操作は行わない
- ChatWorkはアラート通知の受信先としてのみ利用する
- ChatWork API（Webhook or メッセージ送信API）で通知を送る

### 認証・API操作はアプリ側で実施

- Google OAuth認証、Calendar API、Gmail API のすべてをアプリ側で管理する
- ユーザーは初回セットアップ時にGoogleアカウントのOAuth認証を行う
- トークンはDBに暗号化して保存

### 人間承認の挟み込み

- メール送信前に必ずユーザーの承認を挟む（誤送信防止）
- 承認UIはアプリ内チャットに「送信する / 修正する」ボタンを表示
- 将来的に「自動送信モード」を追加する可能性はあるが、初期リリースでは必ず承認を挟む

### 維持費用の抑制

- Google API: 無料枠内での運用を前提
- ChatWork API: フリープランまたは既存契約内
- サーバー: 既存のVercelデプロイメントに統合
- Cron: Vercel Cron Jobs（無料枠: 1日1回、Pro: 1日複数回）

## データモデル（想定）

```prisma
// 日程調整セッション
model ScheduleSession {
  id              String   @id @default(cuid())
  userId          String
  customerName    String   // 顧客名
  customerEmail   String   // 顧客メールアドレス
  status          String   // draft / proposed / confirmed / cancelled
  confirmedSlotId String?  // 確定したスロットのID
  chatHistory     Json     // チャット履歴
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  user            User     @relation(fields: [userId], references: [id])
  slots           ScheduleSlot[]
  emails          ScheduleEmail[]
}

// 候補日スロット
model ScheduleSlot {
  id              String   @id @default(cuid())
  sessionId       String
  startTime       DateTime
  endTime         DateTime
  calendarEventId String?  // Googleカレンダーのイベントid
  status          String   // tentative / confirmed / released

  session         ScheduleSession @relation(fields: [sessionId], references: [id])
}

// メール記録
model ScheduleEmail {
  id              String   @id @default(cuid())
  sessionId       String
  direction       String   // outbound / inbound
  subject         String
  body            String
  gmailMessageId  String?  // GmailのメッセージID
  sentAt          DateTime?
  createdAt       DateTime @default(now())

  session         ScheduleSession @relation(fields: [sessionId], references: [id])
}
```

## API連携詳細

### Google Calendar API

| 操作         | API                                   | 用途                     |
| ------------ | ------------------------------------- | ------------------------ |
| 空き確認     | `freebusy.query`                      | 候補日の抽出             |
| 仮押さえ登録 | `events.insert` (status: tentative)   | 仮予定の作成             |
| 仮押さえ解除 | `events.delete`                       | 不要な仮予定の削除       |
| 確定登録     | `events.update` (status: confirmed)   | 正式スケジュール登録     |

### Gmail API

| 操作       | API                    | 用途                     |
| ---------- | ---------------------- | ------------------------ |
| メール送信 | `messages.send`        | 日程打診メールの送信     |
| 返信検知   | `messages.list` (ポーリング) or `Pub/Sub` | 顧客からの返信検知 |
| メール取得 | `messages.get`         | 返信内容の取得           |

### ChatWork API

| 操作       | API                       | 用途                     |
| ---------- | ------------------------- | ------------------------ |
| 通知送信   | `POST /rooms/{id}/messages` | 未返信アラートの送信   |

## 返信検知の方式検討

| 方式           | メリット                     | デメリット                   | 推奨度 |
| -------------- | ---------------------------- | ---------------------------- | ------ |
| Gmail Pub/Sub  | リアルタイム検知             | セットアップが複雑           | ○      |
| ポーリング     | 実装がシンプル               | API呼び出し頻度の管理が必要  | ◎      |
| Webhook        | リアルタイム                 | Gmailは直接Webhook非対応     | △      |

**初期実装**: ポーリング方式（Vercel Cron Jobs で定期実行）
**将来対応**: Gmail Pub/Sub でリアルタイム化

### ポーリング実装

```
Vercel Cron Job（15分間隔）
  → 進行中のセッションを取得
  → 各セッションの顧客メールアドレスでGmail検索
  → 新着返信があれば解析 → 確定処理
  → 翌営業日超過の未返信があれば → ChatWork通知
```
