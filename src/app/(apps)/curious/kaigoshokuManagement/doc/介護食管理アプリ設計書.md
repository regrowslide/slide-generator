# 介護食管理アプリ 設計書（ユーザーマニュアル）

---

## 第1章: システム概要

### 1.1 目的

本アプリは、介護施設向け食事の受注・製造・配送を一元管理するシステムです。

献立データ（CSV）と受注データ（CSV）を取り込み、製造指示・原材料集計・施設別梱包リストを自動生成します。食事形態（常食・刻み食・極刻み食・ミキサー食）ごとの食数管理と、食事区分（朝食・昼食・昼間食・夕食）ごとの献立管理に対応しています。

### 1.2 業務フロー全体像

```
┌───────────┐     ┌───────────┐     ┌────────────┐     ┌──────────┐     ┌──────────┐
│ 献立CSV   │     │ 受注CSV   │     │ 製造指示   │     │ 原材料   │     │ 梱包     │
│ 取込      │ ──→ │ 取込      │ ──→ │ 自動生成   │ ──→ │ 集計     │ ──→ │ 配送     │
│           │     │           │     │            │     │          │     │          │
│ 献立管理  │     │ 注文管理  │     │ 料理別×    │     │ 材料別   │     │ 施設別   │
│ ページ    │     │ ページ    │     │ 形態別食数  │     │ 必要量   │     │ 食数     │
└───────────┘     └───────────┘     └────────────┘     └──────────┘     └──────────┘
```

### 1.3 ページ構成

本アプリは7つのページで構成されています。

| # | ページ | 説明 |
|---|--------|------|
| 1 | **注文管理** | 受注CSVの取込・確認。施設別・日付別の受注一覧を表示 |
| 2 | **献立管理** | 献立CSVの取込・閲覧。日付×食事区分ごとの料理・材料を管理 |
| 3 | **製造指示** | 献立×受注から料理別の製造数と必要食材を自動計算・表示 |
| 4 | **原材料発注** | 全料理を横断して材料別に必要量を集計・表示 |
| 5 | **梱包・配送** | 施設別×食事区分別の食数をまとめた梱包リストを表示 |
| 6 | **施設管理** | 取引先施設（介護施設等）のマスタ管理 |
| 7 | **マスター管理** | 献立マスタ・施設マスタ等の管理ページ |

### 1.4 データの関係

```
施設マスタ  ─┐
             ├─→  受注（注文管理）  ─┐
食事形態    ─┘                      │
マスタ                              ├─→  製造指示 ─→ 原材料発注
                                    │              ↓
献立（献立管理）  ──────────────────┘         梱包・配送
```

- **献立** と **受注** の2つのCSVデータがベース
- **製造指示**: 献立の料理 × 受注の食数 を掛け合わせて自動計算
- **原材料発注**: 製造指示の食材を材料別に横断集計
- **梱包・配送**: 受注データを施設別にグループ化

## 第2章: 初回セットアップ

### Step 1: 施設マスタの登録

「施設管理」ページで取引先施設を登録します。

| 項目 | 必須 | 説明 |
|------|------|------|
| 施設コード | ○ | 一意のコード（重複チェックあり） |
| 施設名 | ○ | 施設の正式名称 |
| 住所 | | 施設の住所 |
| 電話番号 | | 連絡先電話番号 |
| メールアドレス | | 連絡先メールアドレス |
| 有効/無効 | ○ | 論理削除に使用（無効にすると一覧に表示されない） |

### Step 2: 食事形態マスタの自動投入

食事形態マスタは、受注CSVの初回インポート時に自動で4種類が投入されます。手動登録は不要です。

| コード | 名称 | 色分け |
|--------|------|--------|
| regular | 常食 | 青 |
| kizami | 刻み食 | 緑 |
| kiwamekizami | 極刻み食 | 黄 |
| mixer | ミキサー食 | 紫 |

### Step 3: 献立データの投入

「献立管理」ページから献立CSVをインポートします。詳細は第4章を参照してください。

### Step 4: 受注データの投入

「注文管理」ページから受注CSVをインポートします。詳細は第3章を参照してください。

## 第3章: 注文管理

### 3.1 概要

受注CSVを取り込み、日付別・施設別の受注データを管理するページです。

### 3.2 画面構成

- **ヘッダー**: 取込先施設の選択 + 受注CSV取込ボタン
- **フィルター**: 年月日・施設の選択 + 検索ボタン
- **サマリーカード**: 食事区分ごとの合計食数（朝食・昼食・昼間食・夕食）
- **受注一覧テーブル**: 受注ID・施設・朝食/昼食/夕食の食数

### 3.3 受注CSVの取込

#### CSV形式

```
日付,単位,（常食）朝,（常食）昼,（常食）夜,（刻み食）朝,（刻み食）昼,（刻み食）夜,...
2026/1/31,食,45,50,48,10,12,10,...
2026/2/1,食,42,48,45,8,10,8,...
```

| 列 | 説明 |
|----|------|
| A列 | 日付（`YYYY年M月D日` or `YYYY/MM/DD` 形式） |
| B列 | 単位（固定値「食」） |
| C列以降 | `（食事形態名）朝/昼/夜` の食数 |

#### 取込手順

1. **取込先施設を選択**（任意）: 受注データを紐付ける施設を事前に選択できます。未選択の場合は施設未指定で取り込まれます
2. **「受注取込」ボタン** をクリック
3. CSVファイルを選択
4. インポート結果がログモーダルで表示される

#### 取込時の動作

- 同じ日付・施設・CSV取込の既存受注は**自動削除**されてから再登録される（上書きインポート）
- 全カラムが0の日付行はスキップされる
- 「合計」行は自動でスキップされる
- 食事形態マスタが未登録の場合は自動で初期投入される

### 3.4 受注一覧の操作

- **施設の変更**: 受注一覧の「施設」列のドロップダウンで、受注に紐付く施設を後から変更可能
- **フィルター検索**: 年月日・施設でフィルターし、「検索」ボタンで結果を更新

## 第4章: 献立管理

### 4.1 概要

献立CSVを取り込み、日付ごとの献立データ（料理・材料・栄養素）を管理するページです。

### 4.2 献立の階層構造

献立データは以下の4階層で構成されます。

```
日付
└── Category（食事区分: 朝食、昼食、昼間食、夕食）
    └── Menu（献立: 例「スクランブルエッグ【朝】」）
        └── Dish（料理: 例「ぶどうパン」）
            └── Ingredient（材料: 例「ぶどうパン 60g」）
```

| 階層 | 説明 | 例 |
|------|------|----|
| **Category** | 食事区分（朝食・昼食・昼間食・夕食） | 朝食 |
| **Menu** | 献立の大枠 | スクランブルエッグ【朝】 |
| **Dish** | 個別の料理 | ぶどうパン |
| **Ingredient** | 料理を構成する材料と分量 | ぶどうパン 60g |

### 4.3 献立CSVの取込

#### CSV形式（列マッピング）

| 列 | 内容 |
|----|------|
| B列 | Category（朝食、昼食、昼間食、夕食） |
| C列 | Menu コード |
| D列 | Menu 名前 |
| E列 | Dish コード or Ingredient コード |
| F列 | Ingredient 名前 |
| G列 | 1人分可食量 (g) |
| H列 | エネルギー (kcal) |
| I列 | たんぱく質 (g) |
| J列 | 脂質 (g) |
| K列 | 炭水化物 (g) |
| L列 | ナトリウム (mg) |
| M列 | 食塩 (g) |
| N列 | 野菜量 (g) |

> 3行目（index 2）に日付情報が含まれます。データは8行目（index 7）から開始です。

#### 取込手順

1. **取り込み対象の日付を選択**: 日付入力フィールドで対象日を指定
2. **「CSV取込」ボタン** をクリック
3. CSVファイルを選択
4. インポート結果がログモーダルで表示される

#### 取込時の動作

- 同じ日付の既存献立は**自動削除**されてから再登録される（上書きインポート）
- トランザクション内で一括処理されるため、途中エラー時は全データがロールバックされる
- 「合計」行は自動でスキップされる

### 4.4 献立一覧の閲覧

#### フィルター

| フィルター | 説明 |
|-----------|------|
| 年月 | データが存在する年月のみ選択可能 |
| 日 | 特定日に絞り込み（「全日」で月全体） |
| 食事区分 | 朝食/昼食/昼間食/夕食で絞り込み |
| 献立名検索 | 料理名で部分一致検索 |

#### 一覧テーブルの列

| 列 | 説明 |
|----|------|
| 日付 | 日付と曜日 |
| 区分 | 食事区分（朝食/昼食/昼間食/夕食）をバッジ表示 |
| 献立 | Menu名（献立の大枠名称） |
| 料理コード | Dishのコード |
| 料理名 | Dishの名前 |
| 材料 | 材料数をバッジ表示 |
| 操作 | 「材料一覧」ボタン |

### 4.5 材料一覧の確認

一覧の「材料一覧」ボタンをクリックすると、その料理（Dish）の材料詳細がモーダルで表示されます。

#### 表示内容

- **料理情報**: 料理コード、所属する献立名
- **材料テーブル**: 食品コード、食材名、分量(g)、エネルギー(kcal)、P/F/C
- **栄養素合計（1人前）**: エネルギー、たんぱく質、脂質、炭水化物、食塩相当量、野菜量の合計

## 第5章: 製造指示

### 5.1 概要

献立データと受注データを掛け合わせて、料理別の製造数量と必要食材量を自動計算・表示するページです。

### 5.2 計算ロジック

```
製造食数 = 指定日の受注合計食数（食事区分 × 全形態の合計）
必要食材量 = 材料の1人分量(g) × 製造食数
```

| 項目 | 計算方法 |
|------|---------|
| 料理の製造食数 | その食事区分の全食事形態の受注食数の合計 |
| 形態別食数 | 食事形態ごとの受注食数 |
| 必要食材量 | 各材料の1人分量(g) × 製造食数 |

### 5.3 画面構成

指定日の食事区分ごとにセクションが表示され、各セクション内に料理カードが並びます。

#### 料理カード

| 要素 | 説明 |
|------|------|
| 料理名 | 料理（Dish）の名前 |
| 所属献立名 | 親Menu名（料理名と異なる場合のみ表示） |
| 総製造量 | その料理の総食数 |
| 形態別加工指示 | 常食: ○食、刻み食: ○食、... |
| 必要食材（総量） | 材料名と必要量(g)の一覧 |

### 5.4 操作手順

1. 年月日を選択して「検索」ボタンをクリック
2. 献立データと受注データの両方が登録されている日付を選択
3. 食事区分ごとに料理カードが自動表示される

> **注意**: 献立データまたは受注データが未登録の場合、「この日の献立データはありません」と表示されます。

## 第6章: 原材料発注

### 6.1 概要

指定日の全料理を横断して、材料別に必要量を集計・表示するページです。製造指示の食材データをさらに材料単位で名寄せし、発注に必要な情報をまとめます。

### 6.2 計算ロジック

```
材料の必要量 = Σ(材料の1人分量 × その食事区分の受注食数)
             ※全食事区分（朝食〜夕食）を横断して集計
```

### 6.3 画面構成

| 列 | 説明 |
|----|------|
| 食材コード | 材料の識別コード |
| 食材名 | 材料名 |
| 必要量 | 全料理を合算した総必要量 |
| 単位 | g |
| 使用料理 | どの食事区分のどの料理で使われているかの内訳タグ |

### 6.4 操作手順

1. 年月日を選択して「検索」ボタンをクリック
2. 食材コード順・名前順でソートされた材料一覧が表示される
3. 各材料の「使用料理」タグで、どの料理に使われているかを確認

> **注意**: 献立CSVと受注CSVの両方が取り込まれている必要があります。

## 第7章: 梱包・配送

### 7.1 概要

受注データを施設別にグループ化し、食事区分×食事形態ごとの食数をまとめた梱包リストを表示するページです。

### 7.2 画面構成

施設ごとにカードが表示されます。

#### 施設カード

| 要素 | 説明 |
|------|------|
| 施設名 | 配送先施設の名称 |
| 合計食数 | その施設への総配送食数 |
| 食事区分セクション | 朝食/昼食/夕食ごとの区分 |
| 料理名 | 献立から取得した料理名（献立未登録時は食事区分名） |
| 形態別食数 | 常食: ○食、刻み食: ○食、... |

### 7.3 操作手順

1. 年月日を選択して「検索」ボタンをクリック
2. 施設別のカードが名前順に表示される
3. 各施設の食事区分・形態別の食数を確認

> 施設が受注データに紐付いていない場合は「未登録施設」として表示されます。

## 第10章: 運用フロー（日次業務）

### 10.1 事前準備（月次）

1. **献立CSVの取込**: 栄養士が作成した月間献立CSVを日ごとに取り込む
2. 取込後、「献立管理」ページで料理・材料・栄養素が正しくインポートされたことを確認

### 10.2 日次業務

```
① 受注CSV取込（注文管理）
   ↓
② 受注内容の確認（注文管理）
   ↓
③ 製造指示の確認（製造指示）
   - 料理別の製造食数
   - 形態別の加工指示
   - 必要食材の総量
   ↓
④ 原材料の発注（原材料発注）
   - 材料別に必要量を確認
   - 発注リストとして使用
   ↓
⑤ 梱包・配送準備（梱包・配送）
   - 施設別の配送食数を確認
   - 形態別の食数で梱包準備
```

### 10.3 注意事項

- 献立CSVと受注CSVは**同じ日付**を対象にインポートしてください。日付が一致しないと製造指示・原材料・梱包データが表示されません
- 同じ日付でCSVを再取込すると、既存データは自動で上書きされます
- 施設を受注に紐付けていない場合、梱包・配送ページで「未登録施設」として表示されます

## 第11章: 用語集

| 用語 | 説明 |
|------|------|
| **食事区分** | 1日の食事の時間帯分類。朝食・昼食・昼間食・夕食の4区分 |
| **食事形態** | 食事の加工状態。常食・刻み食・極刻み食・ミキサー食の4種類。利用者の嚥下（えんげ）能力に応じて選択 |
| **常食** | 通常の食事形態。健康な方向けの標準的な調理 |
| **刻み食** | 食材を小さく刻んだ食事形態。咀嚼（そしゃく）力が低下した方向け |
| **極刻み食** | 食材を極めて細かく刻んだ食事形態。刻み食よりさらに細かい |
| **ミキサー食** | 食材をミキサーにかけてペースト状にした食事形態。嚥下困難な方向け |
| **Category** | 献立データの食事区分レベル（朝食・昼食・昼間食・夕食） |
| **Menu** | 献立の大枠。1つのCategoryに複数のMenuが含まれる |
| **Dish** | 個別の料理。1つのMenuに複数のDishが含まれる |
| **Ingredient** | 料理を構成する材料。食品コード・名前・分量・栄養素情報を持つ |
| **製造食数** | ある料理を何食分製造するかの数量。受注食数の合計 |
| **1人分可食量** | 1食分あたりの材料使用量（g） |
| **PFC** | たんぱく質(Protein)・脂質(Fat)・炭水化物(Carbohydrate)の3大栄養素の略称 |
| **食塩相当量** | ナトリウム量から換算した食塩の相当量(g) |
| **施設マスタ** | 取引先の介護施設情報を管理するマスタデータ |
| **食事形態マスタ** | 食事の加工状態（常食・刻み食等）を定義するマスタデータ |
| **上書きインポート** | 同じ日付の既存データを削除してから新規データを登録する取込方式 |
| **論理削除** | データを物理的に削除せず、無効フラグを立てて非表示にする削除方式 |
