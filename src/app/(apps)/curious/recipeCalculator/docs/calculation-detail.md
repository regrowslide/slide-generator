# recipeCalculator 計算ロジック詳細

## 概要

食品製造原価計算システム。レシピの原材料から1パックあたりの製造原価・見積提示価格を算出する。

---

## 1. 原材料コスト計算

各食材について以下の計算を行う。

### 1-1. 重量換算

レシピ記載の単位をkgに変換する。

| 単位   | 変換式                  |
| ------ | ----------------------- |
| kg     | そのまま                |
| g      | ÷ 1000                  |
| L      | そのまま（比重1.0想定） |
| mL, cc | ÷ 1000（比重1.0想定）   |
| 不明   | g扱い（÷ 1000）         |

```
換算重量(kg) = 単位変換(使用量, 単位)
```

### 1-2. 歩留調整単価

歩留まりを考慮した実効単価を算出する。

```
調整後単価 = キロ単価 ÷ (歩留率 ÷ 100)
```

- 歩留 100% → 単価変更なし
- 歩留 80% → 単価は1.25倍（実質2割の可食部ロス）
- 歩留 0% → 0円（ゼロ除算回避）

### 1-3. 食材原価

```
食材原価 = 調整後単価 × 換算重量(kg)
```

### 1-4. 合計値

```
原材料費合計 = 全食材の原価の合計
原材料総重量 = 全食材の換算重量の合計
```

---

## 2. 製造可能重量の決定

2パターンの決定方式がある。**製造可能重量が手動指定されている場合、ロス率は無効**となる。

### A. 手動指定あり

```
製造可能重量(kg) = 製造可能重量(g) ÷ 1000
```

ユーザーが実際の製造可能重量を直接入力する場合。ロス率は無視される。

### B. 自動計算（製造可能重量が空）

```
製造可能重量(kg) = 原材料総重量 × (1 - ロス率 ÷ 100)
```

例: 合計10kg、ロス率5% → 9.5kg

## 3. パック数 / 充填量の計算

2つの入力モードに応じて計算方向が異なる。

### A. 充填量入力モード

ユーザーが1パック充填量を入力 → パック数を自動算出する。

```
パック数 = 切り捨て(製造可能重量(kg) ÷ (充填量(g) ÷ 1000))
```

例: 充填量 200g の場合、9.5kg → 47パック

### B. パック数入力モード

ユーザーがパック数を直接入力 → 充填量を自動算出する。

```
充填量(g) = (製造可能重量(kg) × 1000) ÷ パック数
```

- パック数 50 の場合、9.5kg → 190g/パック
- パック数 0 の場合、充填量は計算しない（無限大防止）

---

## 4. 粗利額の自動提案

再計算実行時、パック数に応じた粗利基準マスタを参照し、条件を満たす場合に粗利額を自動設定する。

### 粗利基準マスタ

| 項目       | 説明                                                     |
| ---------- | -------------------------------------------------------- |
| 食数下限   | この基準が適用されるパック数の下限                       |
| 食数上限   | この基準が適用されるパック数の上限（空の場合は上限なし） |
| 最低粗利額 | パック数レンジに対する最低粗利額（円）                   |
| 最低粗利率 | パック数レンジに対する最低粗利率（%）                    |

### 自動設定の条件

以下の**いずれか**を満たす場合のみ自動設定する:

1. **初回デフォルト値** — 粗利額がデフォルト（200円）のまま
2. **前回の基準値** — 粗利額が前回パック数に対応する基準の最低粗利額と一致

→ ユーザーが手動で粗利を変更した場合は自動設定しない

### 自動設定のフロー

```
1. パック数に該当する粗利基準を検索
2. 条件を満たす場合 → 粗利額を基準の最低粗利額に設定
3. 条件を満たさない場合 → 粗利額は変更しない
```

### 手動復帰（自動セットボタン）

粗利アラートが発生している場合、原価見積パネルの粗利入力欄の横に「自動セット」ボタンが表示される。
クリックすると粗利額を基準の最低粗利額に即座にリセットし、再計算を実行する。

---

## 5. 原価計算

```
原材料費/パック = 原材料費合計 ÷ パック数
製造原価計     = 原材料費/パック + 包材費 + 加工費 + その他費用
見積提示価格   = 製造原価計 + 粗利額
粗利率         = 粗利額 ÷ 見積提示価格 × 100
```

### 計算結果の内訳

| 項目             | 算出方法                                       |
| ---------------- | ---------------------------------------------- |
| 原材料費/パック  | 原材料費合計 ÷ パック数                        |
| 包材費           | 固定入力（円/パック）                          |
| 加工費           | 固定入力（円/パック）                          |
| その他費用       | 固定入力（円/パック）                          |
| **製造原価計**   | 原材料費/パック + 包材費 + 加工費 + その他費用 |
| 粗利額           | 手動入力 または 自動設定（円/パック）          |
| **見積提示価格** | 製造原価計 + 粗利額                            |

---

## 6. 粗利アラート判定

パック数に該当する粗利基準と現在の粗利を比較し、以下の場合に警告を表示する。

```
粗利額が基準未満   = 粗利額 < 最低粗利額
粗利率が基準未満   = 粗利率 < 最低粗利率
警告あり           = 粗利額が基準未満 または 粗利率が基準未満
```

警告時は「自動セット」ボタンが粗利入力欄の横に表示され、最低粗利額へのリセットが可能。

## 7. 計算例

### 入力条件

| 食材     | 使用量 | 単位 | 単価(円/kg) | 歩留(%) |
| -------- | ------ | ---- | ----------- | ------- |
| 鶏むね肉 | 3000   | g    | 1,200       | 85      |
| キャベツ | 2000   | g    | 300         | 90      |
| 調味料A  | 500    | g    | 2,000       | 100     |

製造ロス率: 5%、入力モード: 充填量入力、充填量: 200g
包材費: 30円、加工費: 50円、その他費用: 0円、粗利額: 200円

### ステップ1: 食材原価

| 食材     | 換算重量(kg) | 調整後単価(円/kg)    | 食材原価(円) |
| -------- | ------------ | -------------------- | ------------ |
| 鶏むね肉 | 3.0          | 1,200 ÷ 0.85 = 1,412 | 4,235        |
| キャベツ | 2.0          | 300 ÷ 0.90 = 333     | 667          |
| 調味料A  | 0.5          | 2,000 ÷ 1.00 = 2,000 | 1,000        |

- 原材料費合計 = 5,902円
- 原材料総重量 = 5.5kg

### ステップ2: 製造可能重量

```
製造可能重量 = 5.5 × (1 - 5 ÷ 100) = 5.225kg
```

### ステップ3: パック数

```
パック数 = 切り捨て(5.225 ÷ 0.2) = 26パック
```

### ステップ4: 原価計算

```
原材料費/パック = 5,902 ÷ 26 = 227円
製造原価計     = 227 + 30 + 50 + 0 = 307円
見積提示価格   = 307 + 200 = 507円
粗利率         = 200 ÷ 507 × 100 = 39.4%
```

---

## 8. UI操作フロー

### 製造パラメータパネル

1. 各パラメータはローカル状態で即時反映（入力中は画面のみ更新）
2. 「再計算」ボタン押下で一括サーバー保存 → 全計算を再実行
3. 値が変更されていない場合、再計算ボタンは無効化

### 原価見積パネル

1. 包材費・加工費・その他費用・粗利額を入力
2. 「再計算」ボタンで一括保存 → 全計算を再実行
3. 粗利アラート発生時は「自動セット」ボタンで基準値に即座にリセット可能
