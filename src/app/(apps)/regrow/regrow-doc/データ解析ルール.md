## A. 全項目一覧（データソース別）

### Excelインポート項目（担当者別分析表から自動取得）

| 項目             | Excel上の位置                  | データ型  | 説明                                       |

## B. Excelファイル仕様（パース仕様）

### ファイル命名規則

```
担当者別分析表_asian relaxation villa{店舗名}_{YYYYMMDD}.xlsx
```

- `{店舗名}`: `新潟西店` / `三条店` / `新潟中央店`
- `{YYYYMMDD}`: 集計日（例: `20260209`）

### 店舗名抽出ロジック

1. ファイル名から正規表現 `/villa(.*?)_/` で「villa」と「\_」の間の文字列を取得
2. 取得した文字列に「新潟西」「三条」「新潟中央」が含まれるかで店舗を判別
3. フォールバック: ファイル名全体に店舗名が含まれるかを確認

### ヘッダー情報の取得

| セル位置 | 0-indexed座標 | 内容                                                         |
| -------- | ------------- | ------------------------------------------------------------ |
| B1       | row:0, col:1  | 店舗名（フルネーム）。`"店舗名："` プレフィックスを除去      |
| V1       | row:0, col:21 | 集計期間。`"集計期間："` プレフィックスを除去、`"～"` で分割 |

### 3行パターンのデータ構造

スタッフデータは **row 3（4行目）** から開始し、**3行1セット** で構成されます。

```
行1: [スタッフ名(A)]  [---]  [---]  [---]  [---]  [---]  [売上(G)]  ... [新規客数(W)]
行2: [---]            [---]  [指名数(C)]  [---]  [---]  [---]  [対応客数(G)]
行3: [---]            [---]  [---]  [---]  [---]  [---]  [客単価(G)]
```

具体的なセル位置（0-indexed）:

| データ     | 行オフセット | 列（0-indexed） | 列（アルファベット） |
| ---------- | ------------ | --------------- | -------------------- |
| スタッフ名 | +0           | 0               | A                    |
| 売上合計   | +0           | 6               | G                    |
| 新規客数   | +0           | 22              | W                    |
| 指名数     | +1           | 2               | C                    |
| 対応客数   | +1           | 6               | G                    |
| 客単価     | +2           | 6               | G                    |

### スタッフ名の整形

A列のセル値は `"1位\n田中 美咲"` のようにランキング付きで格納されている。
正規表現 `/^\d+位\n/` でプレフィックスを除去し、`trim()` でスタッフ名を取得する。

### 総合計行の検出ロジック

1. A列を上から走査し、セル値に「総」「合」「計」の3文字すべてが含まれる行を検出
2. スタッフデータの走査中にA列が空になった場合は、そこから5行以内で総合計行を探索
3. 総合計行が見つかった場合:
   - 総合計行 G列（col:6）→ 店舗売上合計
   - 総合計行+1行 G列（col:6）→ 店舗対応客数合計
   - 総合計行+1行 C列（col:2）→ 店舗指名数合計
   - 総合計行+2行 G列（col:6）→ 店舗平均客単価

### 走査の終了条件

- A列のセルが空（`!nameCell?.v`）→ スタッフ走査を終了
- A列のセル値が「総合計」を含む → スタッフ走査を終了、総合計行処理へ
- row が 100 を超える → 安全停止

## C. 計算ロジック詳細

### スタッフ再来率

再来客数 = 対応客数 - 新規客数（W列）で算出する。V列の「失客再来」列は使用しない。

```typescript
// SlidesView.tsx: calculateStaffReturnRate()
const returningCustomers = customerCount - newCustomerCount
const returnRate = Number(((returningCustomers / customerCount) * 100).toFixed(1))
// → 小数点1桁（toFixed(1)）
// customerCount === 0 の場合は 0 を返す
```

### 店舗再来率

```typescript
// SlidesView.tsx: calculateStoreReturnRate(), getStoreMetricFromMonthlyData()
// ManualInputView.tsx: calculateStoreReturnRate()
const totalCustomers = storeRecords.reduce((sum, r) => sum + r.customerCount, 0)
const totalNewCustomers = storeRecords.reduce((sum, r) => sum + r.newCustomerCount, 0)
const returningCustomers = totalCustomers - totalNewCustomers
const rate = Math.round((returningCustomers / totalCustomers) * 100 * 10) / 10
// → 小数点1桁（Math.round で丸め）
// storeRecords が空 または totalCustomers === 0 の場合は 0 を返す
```

### 指名率

```typescript
// SlidesView.tsx: Slide8StaffPerformanceTable
const nominationRate = ((nominationCount / customerCount) * 100).toFixed(1)
// → 小数点1桁（toFixed(1)）
// customerCount === 0 の場合は "-" を表示
```

### CS登録率

```typescript
// SlidesView.tsx: Slide8StaffPerformanceTable
const csRate = ((csRegistrationCount / customerCount) * 100).toFixed(1)
// → 小数点1桁（toFixed(1)）
// customerCount === 0 または csRegistrationCount が未入力の場合は "-" を表示
```

### 店舗CS登録数合計

```typescript
// ManualInputView.tsx: calculateStoreCS()
const total = staffManualData.filter(s => s.storeName === storeName).reduce((sum, s) => sum + (s.csRegistrationCount || 0), 0)
```

### 先月比差分

```typescript
// 売上差分
const salesDiff = staff.sales - (prevStaff?.sales || 0)

// 指名数差分
const nominationDiff = staff.nominationCount - (prevStaff?.nominationCount || 0)

// 再来率差分（小数点1桁）
const returnRateDiff = returnRateCurrent - returnRatePrev
// 表示: returnRateDiff.toFixed(1) + "%"

// 客単価差分
const unitPriceDiff = staff.unitPrice - (prevStaff?.unitPrice || 0)
```

### 丸め方法まとめ

| 計算項目       | 丸め方法                    | 精度      |
| -------------- | --------------------------- | --------- |
| スタッフ再来率 | `Number((...).toFixed(1))`  | 小数点1桁 |
| 店舗再来率     | `Math.round(... * 10) / 10` | 小数点1桁 |
| 指名率         | `(...).toFixed(1)`          | 小数点1桁 |
| CS登録率       | `(...).toFixed(1)`          | 小数点1桁 |
| 再来率差分     | `(...).toFixed(1)`          | 小数点1桁 |

## E. データ保存仕様（localStorage）

```
regrow_data_{YYYY-MM}
```

例: `regrow_data_2026-02`

### 主要な操作関数

| 関数名                   | 説明                          |
| ------------------------ | ----------------------------- |
| `saveMonthlyData`        | YYYY-MM単位でデータを保存     |
| `loadMonthlyData`        | YYYY-MM単位でデータを読み込み |
| `getAllMonths`           | 全YYYY-MMリストを取得（降順） |
| `deleteMonthlyData`      | YYYY-MM単位でデータを削除     |
| `createEmptyMonthlyData` | 空のMonthlyDataを生成         |
| `getCurrentYearMonth`    | 現在の月をYYYY-MM形式で取得   |
| `getPreviousMonth`       | 前月のYYYY-MMを算出           |
| `getNextMonth`           | 翌月のYYYY-MMを算出           |
| `formatYearMonth`        | YYYY-MM → "YYYY年M月" に変換  |

### マイグレーション

旧構造（`staffUtilization` / `staffEvaluation`）から新構造（`staffManualData`）への自動変換が `loadMonthlyData` に組み込まれている。読み込み時に旧フィールドを検出した場合、新構造に変換してlocalStorageに再保存する。

### データフロー

```
ユーザー操作（入力/インポート）
  → DataContext の update 関数呼び出し
    → monthlyData ステートを更新
      → saveMonthlyData() で localStorage に保存
```

月切り替え時:

```
currentYearMonth 変更
  → useEffect で loadMonthlyData(newMonth) 実行
    → monthlyData ステートを更新
      → 画面に反映
```

モックデータ優先:

```
getMonthlyData(yearMonth)
  → MOCK_DATA[yearMonth] が存在すれば優先的に返却
    → 存在しなければ loadMonthlyData(yearMonth) にフォールバック
```
