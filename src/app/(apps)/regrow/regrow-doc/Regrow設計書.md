# Regrow ユーザーマニュアル（設計書）

> 最終更新: 2026-03-04

---

## 第1章 アプリ概要

### 1.1 目的

Regrowは、リラクゼーションサロンの**月次業績レポートを自動生成する**ためのアプリです。

毎月のExcel「担当者別分析表」と、手動で入力するKPIデータを組み合わせ、14枚のスライド資料（グラフ・テーブル付き）を自動で作成します。

### 1.2 できること

| 機能 | 説明 |
|------|------|
| Excelインポート | 担当者別分析表を取り込み、スタッフ売上・客数等を自動抽出 |
| 手動データ入力 | 店舗KPI（稼働率・コメント等）やスタッフ個別データを入力 |
| スライド資料閲覧 | 14枚構成のレポートスライドを自動生成・閲覧 |
| マスタ管理 | 店舗・ユーザー・権限の管理 |
| 月別データ管理 | YYYY-MM単位でデータを管理し、過去月の閲覧・比較が可能 |

### 1.3 画面構成

```
Regrow
├── レポート画面（/regrow/report）
│   ├── Excel取込          ← Excelファイルのアップロード
│   ├── データ確認          ← インポート済みデータの一覧表示
│   ├── 手動入力            ← 店舗KPI・スタッフ稼働率等の入力
│   └── スライド            ← 14枚のスライド資料を閲覧
│
└── マスタ管理画面（/regrow/master）
    ├── 店舗マスタ          ← 店舗の追加・編集・削除
    ├── ユーザー・権限管理  ← ユーザーの追加・担当店舗割当・有効/無効切替
    └── 権限割当            ← 管理者/店長/閲覧者の権限設定
```

### 1.4 権限（ロール）

| ロール | 説明 | できること |
|--------|------|-----------|
| 管理者（admin） | システム管理者 | 全機能の操作、マスタ管理、データ編集 |
| 店長（manager） | 店舗責任者 | データの閲覧 |
| 閲覧者（viewer） | 一般スタッフ | スライドの閲覧のみ |

- 管理者のみがExcel取込・データ確認・手動入力タブを操作できます
- スライドタブは全ロールで閲覧可能です

## 第2章 初回セットアップ

### 2.1 店舗を登録する

1. **マスタ管理画面** を開く
2. 「店舗マスタ」タブを選択
3. 「新規追加」ボタンをクリック
4. 以下を入力してモーダルで「追加」をクリック

| 項目 | 必須 | 説明 | 入力例 |
|------|------|------|--------|
| 名前 | ○ | 店舗の略称（レポートやスライドに表示される名前） | 港北店 |
| フルネーム | - | 店舗の正式名称 | Relaxation Salon SAMPLE港北店 |
| 有効 | - | スイッチON/OFF（デフォルトON） | ON |

### 2.2 ユーザーを登録する

1. 「ユーザー・権限管理」タブを選択
2. 「新規追加」ボタンをクリック
3. 以下を入力

| 項目 | 必須 | 説明 |
|------|------|------|
| 名前 | ○ | ユーザーの表示名（Excel上のスタッフ名と一致させる） |
| メールアドレス | - | ログイン用（省略可） |
| パスワード | - | ログイン用（省略可） |

> **重要**: ユーザー名はExcelインポート時のスタッフ名マッチングに使用されます。Excelの担当者名と完全一致する名前で登録してください。

### 2.3 担当店舗を設定する

ユーザー一覧の「担当店舗」ドロップダウンから、そのユーザーの**現在の所属店舗**を選択します。

- 店舗異動があった場合は、担当店舗を新しい店舗に変更してください
- 過去にインポートしたデータはレコード内に店舗情報を保持しているため、異動後も当時の店舗で表示されます
- 担当店舗が未設定のユーザーもインポート時のマッチング対象になります

### 2.4 権限を設定する

1. 「権限割当」タブを選択
2. 権限割当テーブルで、各ユーザーに対してロール（admin / manager / viewer）を割り当てる

### 2.5 対象月を作成する

1. レポート画面を開く
2. 画面上部の「対象月」エリアで「+ 新規作成」をクリック
3. YYYY-MM形式（例: 2026-03）を入力して「作成」をクリック
4. 空のデータが初期化されます

## 第3章 Excel取込（インポート）

### 3.1 対象ファイル

担当者別分析表のExcelファイル（.xlsx / .xls）が対象です。

```
例: 担当者別分析表_Relaxation Salon SAMPLE港北店_20260301.xlsx
```

### 3.2 操作手順

1. レポート画面の「Excel取込」タブを開く
2. **STEP 1**: 取込先の「年月」と「店舗」を選択
3. **STEP 2**: ファイルをドラッグ＆ドロップ、またはクリックしてファイルを選択
4. プレビュー画面が表示される
5. スタッフのマッチング結果を確認
6. 問題がなければ「確認して取り込む」をクリック

### 3.3 プレビュー画面の見かた

プレビュー画面では以下が表示されます。

- **取込先情報**: 選択した年月 / 店舗 / 集計期間
- **店舗合計**: 売上合計・対応客数・指名数・客単価
- **スタッフ一覧**: 各スタッフの売上データとマッチング状況

#### マッチング状態の表示

| アイコン | 状態 | 説明 |
|----------|------|------|
| ✅ | DB登録済 | ユーザー名と一致するユーザーが1名見つかった |
| ⚠️ | 未登録 | ユーザー名と一致するユーザーがいない |
| 🔄 | 同名ユーザー（未選択） | 同じ名前のユーザーが複数いるため選択が必要 |

#### 同名ユーザーがいる場合

同じ名前のユーザーが複数登録されている場合、該当スタッフ行にドロップダウンが表示されます。

1. ドロップダウンから正しいユーザーを選択（ユーザー名と現在の担当店舗が表示されます）
2. すべての同名ユーザーを選択し終えると「確認して取り込む」ボタンが有効になります

### 3.4 取込時のルール

- 同じ店舗・年月のデータを再度アップロードすると**上書き**されます
- 未登録スタッフがいる場合はインポートできません（先にマスタ管理でユーザー登録が必要）
- ユーザーのマッチングは**名前のみ**で行います（店舗は関係ありません）

### 3.5 Excelから抽出される項目

| 項目 | Excel上の位置 | 説明 |
|------|--------------|------|
| 店舗名（フル） | B1セル | 「店舗名：」の後の文字列 |
| 集計期間 | V1セル | 開始日～終了日 |
| スタッフ名 | A列（4行目～） | 順位番号を除いた名前 |
| 売上合計 | G列（1行目） | スタッフの月間売上 |
| 新規客数 | W列（1行目） | 新規のお客様数 |
| 対応客数 | G列（2行目） | 対応した総客数 |
| 指名数 | C列（2行目） | 指名された件数 |
| 客単価 | G列（3行目） | 1人あたりの単価 |

※ スタッフデータは3行で1組（売上→客数→客単価）の構造です。

## 第4章 データ確認

### 4.1 概要

「データ確認」タブでは、インポート済みのExcelデータを店舗別に一覧表示できます。

### 4.2 操作

1. 画面上部の店舗タブで確認したい店舗を選択
2. スタッフ一覧がランキング順で表示される

### 4.3 表示項目

| 列 | 説明 |
|----|------|
| 順位 | Excel上のランキング |
| 担当者名 | スタッフ名 |
| 売上合計 | 月間売上金額 |
| 客数 | 対応客数 |
| 指名数 | 指名件数 |
| 客単価 | 1人あたりの平均単価 |

- テーブル最下行に**店舗合計**が表示されます
- インポート日時とレコード数も確認できます
- 未取込の店舗には「（未取込）」と表示されます

## 第5章 手動入力

### 5.1 概要

Excelから自動取得できないデータを手動で入力する画面です。3つのタブに分かれています。

- **店舗KPI**: 店舗単位の月次指標
- **スタッフ稼働率・CS登録数**: スタッフ個人の稼働率とCS登録数
- **お客様の声**: お客様からのフィードバック

### 5.2 店舗KPIタブ

各店舗について以下が表示されます。

| 項目 | 入力 / 自動 | 説明 |
|------|------------|------|
| 稼働率 (%) | **自動計算** | スタッフ稼働率の平均値（読み取り専用） |
| 再来率 (%) | **自動計算** | (対応客数 - 新規客数) / 対応客数 × 100（読み取り専用） |
| CS登録数 | **自動計算** | スタッフCS登録数の合計（読み取り専用） |
| コメント | 手動入力 | 月次の店舗振り返りコメント |

> 稼働率・再来率・CS登録数は自動計算のため、直接入力できません。スタッフ稼働率・CS登録数タブでスタッフ個別のデータを入力すると自動で反映されます。

### 5.3 スタッフ稼働率・CS登録数タブ

Excelをインポートすると、スタッフ一覧が自動で表示されます。各スタッフに対して以下を入力します。

| 項目 | 入力 / 自動 | 説明 |
|------|------------|------|
| 稼働率 (%) | 手動入力 | スタッフ個人の月間稼働率 |
| 再来率 (%) | **自動計算** | (対応客数 - 新規客数) / 対応客数 × 100（読み取り専用） |
| CS登録数 | 手動入力 | スタッフのCS（カスタマーサービス）登録件数 |

### 5.4 お客様の声タブ

- テキストエリアにお客様からのフィードバックを自由に入力
- この内容はスライド14「お客様の声」にそのまま反映されます

### 5.5 自動保存

- 入力欄からフォーカスが外れた時点で**自動保存**されます（保存ボタンはありません）
- 保存時に画面右上に「保存完了 ✅」が一時表示されます
- データはデータベースに直接保存されます

### 5.6 自動計算の計算式

| 項目 | 計算式 |
|------|--------|
| スタッフ再来率 | (対応客数 − 新規客数) ÷ 対応客数 × 100（小数第1位まで） |
| 店舗再来率 | 店舗内全スタッフの再来客数合計 ÷ 対応客数合計 × 100 |
| 店舗稼働率 | 店舗内全スタッフの稼働率の平均値（小数第1位まで） |
| 店舗CS登録数 | 店舗内全スタッフのCS登録数の合計 |

### 5.7 閲覧専用モード

管理者以外のロールでは、手動入力画面は「閲覧専用」と表示され、データの編集はできません。

## 第6章 スライド閲覧

### 6.1 概要

ExcelデータとKPIデータから自動生成された14枚のスライド資料を閲覧する画面です。

### 6.2 スライド構成（全14枚）

| No. | タイトル | 内容 |
|-----|----------|------|
| 1 | タイトルスライド | 月次業績レポート、対象月の表示 |
| 2 | 目次 | レポートの構成一覧 |
| 3 | 全体サマリー | 店舗別KPIテーブル + 店舗コメント |
| 4 | 客単価 - 年間推移 | 3店舗の客単価を月別グラフで比較 |
| 5 | 稼働率 - 年間推移 | 3店舗の稼働率を月別グラフで比較 |
| 6 | 再来率 - 年間推移 | 3店舗の再来率を月別グラフで比較 |
| 7 | 全指標統合 - 年間推移 | 客単価・稼働率・再来率を1つのグラフに統合表示 |
| 8 | スタッフ別パフォーマンス | スタッフ別テーブル（売上・稼働率・指名割合・再来率・CS登録率） |
| 9 | スタッフ稼働率チャート | スタッフ稼働率の縦棒グラフ |
| 10 | スタッフ別先月比 (1) テーブル | 売上金額・指名件数の今月/先月/差分テーブル |
| 11 | スタッフ別先月比 (1) グラフ | 売上（棒グラフ）+ 指名（折れ線）の複合グラフ |
| 12 | スタッフ別先月比 (2) テーブル | 再来率・客単価の今月/先月/差分テーブル |
| 13 | スタッフ別先月比 (2) グラフ | 客単価（棒グラフ）+ 再来率（折れ線）の複合グラフ |
| 14 | お客様の声 | 手動入力されたお客様のフィードバックを表示 |

### 6.3 閲覧モード

画面上部のコントロールバーで3つの閲覧モードを切り替えられます。

| モード | 説明 |
|--------|------|
| スクロール | 全14枚を縦に並べてスクロール表示（デフォルト） |
| ページ切替 | 1枚ずつ表示し、矢印キーまたはボタンで前後移動 |
| 全画面 | ブラウザ全画面でスライドを表示 |

#### ページ切替モードの操作

- **前のスライド**: 画面左側をクリック / ← キー
- **次のスライド**: 画面右側をクリック / → キー
- スライド番号インジケータが画面下部に表示されます

### 6.4 店舗フィルタ

- スライド3～13に適用されるフィルタです
- 各店舗のチェックボックスで表示ON/OFFを切り替えられます
- 複数店舗を同時に表示可能

### 6.5 スタッフフィルタ

- スタッフ別のスライド（8～13）で使用可能
- 表示したいスタッフをチェックボックスで選択
- 店舗ごとにまとめて表示されます

### 6.6 累計平均との比較

- 年間推移グラフ（スライド4～6）で、累計平均の折れ線を重ねて表示する機能があります
- トグルで切り替え可能

### 6.7 印刷 / PDF出力

ブラウザの印刷機能（Ctrl+P / Cmd+P）を使って印刷またはPDF出力ができます。

## 第7章 マスタ管理

### 7.1 アクセス方法

サイドメニューから「マスタ管理」を選択します。

### 7.2 店舗マスタ

店舗の登録・編集・削除を行います。

#### 店舗一覧テーブル

| 列 | 説明 |
|----|------|
| ID | 店舗の識別番号（自動採番） |
| 名前 | 店舗の略称（スライドに表示される名前） |
| フルネーム | 店舗の正式名称 |
| 状態 | 有効 / 無効 |
| 操作 | 編集・削除ボタン |

#### 操作

- **新規追加**: 「新規追加」ボタン → 名前・フルネーム・有効/無効を入力
- **編集**: 鉛筆アイコンをクリック → モーダルで編集
- **削除**: ゴミ箱アイコンをクリック → 確認ダイアログ後に削除

### 7.3 ユーザー・権限管理

ユーザーの登録・担当店舗の割当・有効/無効の管理を行います。

#### ユーザー一覧テーブル

| 列 | 説明 |
|----|------|
| 名前 | ユーザーの表示名 |
| 担当店舗 | ドロップダウンで店舗を選択（未設定も可） |
| 状態 | 有効/無効のスイッチ |
| 操作 | 削除ボタン |

#### 操作

- **新規追加**: 「新規追加」ボタン → 名前・メール・パスワードを入力
- **担当店舗変更**: ドロップダウンで店舗を選択（即時反映）
- **有効/無効切替**: スイッチをクリック（無効ユーザーはインポート時のマッチング対象外）
- **削除**: ゴミ箱アイコン → 確認ダイアログ後に完全削除

> **注意**: ユーザーを削除すると、過去データのスタッフ紐付けが解除されます。退職者は「無効」にすることを推奨します。

### 7.4 権限割当

権限割当テーブルで、各ユーザーにロール（admin / manager / viewer）を設定します。

## 第8章 月の管理

### 8.1 対象月の切り替え

レポート画面上部の「対象月」ドロップダウンで、データを表示する月を切り替えます。

- 最新の月がデフォルトで選択されます
- ドロップダウンにはデータが存在する全月が一覧表示されます

### 8.2 新規月の作成

1. 「+ 新規作成」ボタンをクリック
2. YYYY-MM形式で年月を入力（例: 2026-04）
3. 「作成」をクリック
4. 空のデータが初期化され、新しい月が選択されます

> 既に存在する月は作成できません（重複チェックあり）。

---

## 第9章 データ管理（管理者向け）

### 9.1 データリセット

マスタ管理画面の「データリセット」ボタンで、Regrowアプリの全データを削除できます。

- 店舗データ、ユーザーデータ、月次レポートデータがすべて削除されます
- **この操作は取り消せません**
- 確認ダイアログが表示されます

### 9.2 シードデータ投入

マスタ管理画面の「シードデータ投入」ボタンで、デモ用のサンプルデータを投入できます。

- 既存データをリセットした上でサンプルデータが投入されます
- 3店舗 × 各5名のスタッフデータが作成されます
- 12ヶ月分の月次データ（季節変動あり）が生成されます

---

## 第10章 月次レポート作成の全体フロー

毎月のレポート作成は以下の手順で行います。

```
① 新しい月を作成（例: 2026-04）
    ↓
② 各店舗のExcelファイルをインポート（店舗数分繰り返し）
    ↓
③ 手動入力
   - スタッフ稼働率・CS登録数を入力
   - 店舗コメントを入力
   - お客様の声を入力
    ↓
④ スライドを確認
    ↓
⑤ 必要に応じて印刷 / PDF出力
```

### 完了判定

| ステップ | 完了条件 |
|----------|----------|
| Excel取込 | 全店舗分のデータがインポートされている |
| 手動入力 | 全店舗のKPI + 1件以上のスタッフデータが入力されている |
| スライド | Excel取込と手動入力の両方が完了している |

手動入力が未完了の場合、レポート画面のタブに赤い **「!」** バッジが表示されます。

## 用語集

| 用語 | 説明 |
|------|------|
| 担当者別分析表 | 業務システムから出力されるExcelファイル。スタッフごとの売上・客数等が記載されている |
| 稼働率 | スタッフまたは店舗の月間稼働率（%）。スタッフ稼働率の平均が店舗稼働率になる |
| 再来率 | リピーターの割合。(対応客数 − 新規客数) ÷ 対応客数 × 100 で算出 |
| 客単価 | お客様1人あたりの平均売上金額 |
| 指名数 | お客様から指名を受けた件数 |
| CS登録数 | カスタマーサービスへの登録件数 |
| 先月比 | 当月と前月のデータの差分。プラスは増加、マイナスは減少 |
| 累計平均 | 当年1月から当月までの平均値 |
