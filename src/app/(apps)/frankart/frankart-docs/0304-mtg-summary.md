# フランクアート社 MTG議事録まとめ（2026/03/04）

## 業務全体を統合したシステムのイメージ・内容

### 現状の課題

- **ツールが分散**: スプレッドシート、チャットワーク、kintone（3月末終了）、弥生会計（ローカル版）など複数ツールに情報が散在
- **手作業が多い**: 見積書・請求書はスプレッドシートで手打ち、案件ごとにシートを複製・編集
- **前任者の開発が途中で止まっている**: GAS連携やkintoneエクスポート機能が未完成のまま
- **業務フローが標準化されていない**: マニュアル化・ルール化が不十分

---

### 統合システムの目指す姿

**案件を中心とした一元管理システム**（Webシステム）

| 機能 | 内容 |
|---|---|
| **案件管理** | 案件一覧を全社員が閲覧可能。顧客名・開始日・担当営業・協力会社など共通情報を一元管理 |
| **案件別チャット** | チャットワークのルームに相当する、案件専用のコミュニケーション機能 |
| **進捗管理** | 営業の動き・進捗報告をシステム上で入力・共有（掲示板的な役割） |
| **書類作成・発行** | 見積書・請求書・発注書・契約書を案件データから自動生成し、PDF出力→Googleドライブ保存 |
| **ファイル管理** | 案件に紐づくファイルの保存・参照。履歴からGoogleドライブへ直接ジャンプ可能 |
| **権限管理** | 協力会社には関係する案件のみ表示。ユーザーごとに閲覧・編集権限を制御 |
| **売上・集計** | 案件ごとの売上入力→月次集計・レポート作成 |
| **外部連携（将来）** | Googleカレンダー連携、Gmail通知、弥生会計向けCSVエクスポート、Googleスライドでの月次報告資料自動生成 |

---

### 進め方の方向性（MTGで合意した内容）

1. **まず社内の業務整理・標準化を進める**（途中ゴールの設定）
2. フランクアート社内で要件を整理した上で、改めて相談の場を設ける
3. **フェーズを切って段階的に開発**（例: 第1フェーズ40〜50万の予算感でミニマル版 → 追加フェーズで機能拡張）
4. デモ先行開発で実物を見ながら要件を固める方式

---

### 技術選定に関する議論のポイント

- **Zapier vs GAS**: Google系で完結するならGASの方が柔軟・低コスト。AIの活用でGASの学習コストも下がっている
- **Backlog等の既存SaaS**: 7〜8割はカバーできるが、請求書発行や高度な権限管理などで痒いところが残る
- **スプレッドシート単体の限界**: 協力会社への権限制御・ユーザー別表示切替はスプレッドシートでは困難 → Webシステムが必要
