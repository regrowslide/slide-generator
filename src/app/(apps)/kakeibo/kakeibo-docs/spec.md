# 家計簿アプリ 仕様書

## 1. 概要

### 1.1 アプリの目的

家計簿が続かない問題を解決するため、**モバイルファーストで入力のしやすさを最優先**にした家計簿アプリ。日々の収支入力から年間推移、ライフプラン、資産推移まで一貫して管理できる。

### 1.2 対象ユーザー

- 夫婦（家族）で共有利用を想定
- 既存の `User` モデル + `UserRole`（`kakeibo-admin` / `kakeibo-viewer`）で権限管理

### 1.3 技術スタック

- Next.js App Router（既存マルチアプリ基盤 `/src/app/(apps)/kakeibo/`）
- Prisma（PostgreSQL）
- shadcn UI + Tailwind CSS
- Server Actions + サービスクラス

## 3. データモデル

### 3.1 カテゴリマスタ（KbCategory）

家計簿の費目を管理する。

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int（PK） | 自動採番 |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |
| userId | String（FK → User） | 所有ユーザー |
| name | String | カテゴリ名（例：「食費」「パパ給与」） |
| icon | String? | 絵文字アイコン（例：🍚、💰） |
| division | String | 区分（後述の enum 参照） |
| weeklyBudget | Int? | 週予算（円） |
| monthlyBudget | Int? | 月間予算（円） |
| sortOrder | Float | 並び順 |
| active | Boolean | 有効フラグ |
| isFavorite | Boolean | よく使うカテゴリフラグ（上位3件をタップ式表示） |

**区分（division）の値:**

| 値 | 説明 |
|----|------|
| `INCOME` | 収入 |
| `SAVING` | 先取り貯金・投資 |
| `FIXED_EXPENSE` | 支出（固定費） |
| `VARIABLE_EXPENSE` | 支出（変動費） |
| `SPECIAL_EXPENSE` | 支出（特別費） |

### 3.2 支払方法マスタ（KbPaymentMethod）

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int（PK） | 自動採番 |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |
| userId | String（FK → User） | 所有ユーザー |
| name | String | 名称（例：「楽天カード」「現金」） |
| withdrawalDay | Int? | 引落日（1〜31。null＝即時/現金） |
| withdrawalAccount | String? | 引落口座名 |
| sortOrder | Float | 並び順 |
| active | Boolean | 有効フラグ |

### 3.3 収支レコード（KbTransaction）

日々の収支入力を保存する中心テーブル。

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int（PK） | 自動採番 |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |
| userId | String（FK → User） | 入力ユーザー |
| date | DateTime | 取引日（UTC保存） |
| categoryId | Int（FK → KbCategory） | カテゴリ |
| paymentMethodId | Int?（FK → KbPaymentMethod） | 支払方法（収入の場合は入金口座として使用） |
| amount | Int | 金額（円） |
| satisfaction | String? | 満足度（`GOOD` / `NEUTRAL` / `BAD`） |
| memo | String? | 備考 |
| weekNumber | Int? | その月の第何週か（1〜5。変動費の週予算管理用） |

**満足度の値と表示:**

| 値 | 表示 |
|----|------|
| `GOOD` | 〇 |
| `NEUTRAL` | △ |
| `BAD` | ✕ |

## 4. 各画面の詳細仕様

### 4.1 収支入力（メイン画面）

**パス:** `/kakeibo`

**設計思想:** モバイルファーストで片手操作を想定。最小タップ数で入力完了できること。

#### 画面構成

1. **収支切替タブ**
   - 「支出」「収入」のトグル切替
   - 初期値：支出

2. **日付選択**
   - 初期値：今日
   - カレンダーピッカーまたは左右スワイプで日付変更

3. **支払方法選択**
   - 初期値：マスタの sortOrder 先頭
   - 横スクロールのチップ式 UI

4. **カテゴリ選択**
   - よく使う上位3件（`isFavorite = true`）をアイコン付きタップボタンで大きく表示
   - それ以外はプルダウン（セレクトボックス）で選択
   - 各カテゴリに絵文字アイコンを表示
   - 収支切替に連動して表示カテゴリをフィルタ（収入 → `INCOME` のみ、支出 → `FIXED_EXPENSE` / `VARIABLE_EXPENSE` / `SPECIAL_EXPENSE`）

5. **金額入力**
   - 必須
   - テンキー式の数値入力 UI
   - フォーカス時にキーボード自動表示

6. **満足度（任意）**
   - 〇 / △ / ✕ の3ボタン
   - 支出時のみ表示、未選択可

7. **備考（任意）**
   - 1行テキスト入力

8. **登録ボタン**
   - 押下後に収支レコードを保存

#### 登録後の動作

- **予算残ポップアップ表示:**
  - 週予算の残り：「今週の食費 残り ¥42,000」
  - 月予算の残り：「今月の食費 残り ¥150,000」
  - 予算超過時は赤字で警告表示
- ポップアップは3秒後に自動クローズ、またはタップで閉じる
- 入力フォームはリセットされ、次の入力にすぐ移れる状態にする

### 4.3 年間推移シート

**パス:** `/kakeibo/annual`

#### 画面構成

- **年度選択**（上部）
- **費目×12ヶ月の表形式**
  - 行：カテゴリ（区分ごとにグループ化。収入 → 固定費 → 変動費 → 特別費の順）
  - 列：1月〜12月 + 平均 + 年間合計
  - 各セル：実績金額（大）/ 予算金額（小・グレー）
  - **色判定:** 予算内＝緑系、予算超過＝赤系
- **区分ごとの小計行**
- **支出率表示**（収入に対する各区分の割合）
- **特別費セクション:**
  - 前年繰越額の表示
  - 特別費の残り予算を自動計算して表示

#### モバイル対応

- 横スクロール対応のテーブル
- 月タップで該当月の詳細表示

---

### 4.4 収支可視化シート

**パス:** `/kakeibo/summary`

#### 画面構成

1. **月平均セクション（毎月発生するもの）**
   - 収入合計 / 先取り貯金合計 / 固定費合計 / 変動費合計
   - 各区分の内訳（カテゴリ別金額）

2. **年計セクション（臨時）**
   - 特別費等の年間集計

3. **今月の収支サマリー**
   - 収入 − 先取り − 支出 ＝ 余り貯金
   - 貯蓄合計（先取り + 余り貯金）
   - 支出割合（円グラフまたはドーナツチャート）
   - 貯蓄率（%）

#### 可視化

- 収入/先取り/固定費/変動費の内訳を棒グラフまたはドーナツチャートで表示

---

### 4.5 特別費管理シート

**パス:** `/kakeibo/special`

#### 画面構成

1. **予算セクション（上部）**
   - 前年繰越額（`KbSpecialBudget.carryoverAmount`）
   - 今年の積立金額（`KbSpecialBudget.annualSavingAmount`）
   - 予算合計＝前年繰越 + 積立金額

2. **月別×特別費カテゴリ テーブル**
   - 行：特別費カテゴリ（`division = SPECIAL_EXPENSE` のカテゴリ）
   - 列：1月〜12月
   - 各セル：予算 / 実績

3. **残り予算**
   - 特別費の残り＝予算合計 − 実績合計
   - カテゴリごとの残り予算も表示

  - 各セル：合計金額
- **各支払方法の詳細情報**

  - 引落口座

### 4.7 満足度振り返り

**パス:** `/kakeibo/satisfaction`

- **対象月の選択**（月送りUI）
- **購入品一覧**

#### フィルタ機能

### 4.8 ライフプランシート

**パス:** `/kakeibo/lifeplan`

#### 画面構成

1. **上部：家族年齢推移**
   - 行：家族メンバー（`KbFamilyMember`）+ イベント行（手動入力）
   - 列：年（2025〜2082 等、設定可能な範囲）
   - 各セル：その年の年齢を自動計算

2. **下部：収支カテゴリ行**
   - 区分ごとにグループ化（収入 → 固定費 → 変動費 → 特別費）
   - 各行の設定：
     - 初期値（年額）
     - 算定期間（複数設定可、`KbLifePlanPeriod`）
     - 上昇率（%）
   - 各セル：その年の予測金額を自動計算（初期値 × (1 + 上昇率)^年数）

3. **年間貯金額行**
   - 収入合計 − 支出合計を自動計算

#### 機能

- **「平均値から引用」ボタン:**
  - 年間入力の平均値 × 12 を自動計算して baseAmount にセット
- **非表示機能:**
  - 項目を非表示にすると末尾の「場外」セクションに移動
  - いつでも表示状態に戻せる
- **任意項目の追加:**
  - カテゴリに紐づかない自由入力行を追加可能

#### 上昇率のデフォルト値（参考）

| 区分 | デフォルト上昇率 |
|------|------------------|
| 収入（給与等） | 1.5% |
| 固定費 | 0% |
| 変動費 | 0.5% |
| 特別費 | 0% |

---

### 4.9 資産推移シート

**パス:** `/kakeibo/assets`

#### 画面構成

1. **資産項目一覧**（`KbAssetItem`）
   - 資産種別ごとにグループ化
   - 各項目：名前 / 現在残高 / 利率 / 毎月積立額

2. **推移テーブル**
   - 行：資産項目
   - 列：年（ライフプランと同じ範囲）
   - 各セル：予測残高を自動計算
     - 計算式：前年残高 × (1 + 利率) + 年間積立額

3. **資産種別ごとの小計行**
   - 現金合計 / 投資合計 / 不動産合計 / その他合計 / 負債合計

4. **純資産行**
   - （現金 + 投資 + 不動産 + その他） − 負債

#### 資産種別

| assetType | 説明 | デフォルト利率 |
|-----------|------|----------------|
| `CASH` | 現金・預金 | 0.001% |
| `INVESTMENT` | 投資資産（投資信託・暗号資産・iDeCo・企業型DC等） | 4.0% |
| `REAL_ESTATE` | 不動産 | -1.0% |
| `OTHER` | その他資産 | 0% |
| `LIABILITY` | 負債（住宅ローン等） | 0% |

#### 機能

- 項目の任意追加・削除
- 利率・積立額の個別設定
- グラフ表示（折れ線チャート）

---

### 4.10 カレンダー

**パス:** `/kakeibo/calendar`

#### 画面構成

1. **ノーマネーデーサマリー（上部）**
   - 今月のノーマネーデー日数
   - 最長連続ノーマネーデー記録

2. **月カレンダー（メイン）**
   - 月曜始まりの7列グリッド（月〜日）
   - 各セル:
     - **支出あり**: 金額を赤字で表示
     - **ノーマネーデー**: 🌸を中央表示、薄緑背景
     - **未来の日付**: グレーアウト（判定対象外）
   - 土曜は青文字、日曜は赤文字
   - 今日の日付にemeraldボーダー

3. **日付タップモーダル**
   - タップでその日のトランザクション一覧を表示
   - カテゴリ名、金額、支払方法、満足度を表示

#### ノーマネーデー判定ロジック

- 「支出」カテゴリ（`fixed_expense`, `variable_expense`, `special_expense`）のトランザクションが0件の日
- 収入や先取り貯金のみの日はノーマネーデー扱い
- 未来の日付は判定対象外

---

### 4.11 マスタ管理

**パス:** `/kakeibo/admin`

#### タブ構成

1. **カテゴリ管理**
   - カテゴリの一覧表示（区分ごとにグループ化）
   - 追加 / 編集 / 削除（論理削除＝active: false）
   - ドラッグ&ドロップで並び替え
   - よく使うフラグの切替（最大3件）
   - 設定項目：名称 / アイコン（絵文字ピッカー） / 区分 / 週予算 / 月間予算

2. **支払方法管理**
   - 支払方法の一覧表示
   - 追加 / 編集 / 削除（論理削除）
   - ドラッグ&ドロップで並び替え
   - 設定項目：名称 / 引落日 / 引落口座

3. **家族メンバー管理**
   - 家族メンバーの一覧
   - 追加 / 編集 / 削除
   - 設定項目：名前 / 生まれ年

## 7. 非機能要件

### パフォーマンス

- 収支入力画面の初回表示：1秒以内
- 入力〜保存〜ポップアップ表示：0.5秒以内

### セキュリティ

- 全データは `userId` でフィルタし、他ユーザーのデータにアクセス不可
- Server Actions でのバリデーション必須

### レスポンシブ対応

- モバイル（375px〜）を最優先で設計
- タブレット・PCでも閲覧可能
- 集計テーブルは横スクロール対応
