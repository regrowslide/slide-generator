# 山の会（KCAC）システム 機能要件定義書

---

## 1. プロジェクト概要

### 1.1 プロジェクト名

山の会（KCAC）情報一元管理システム

### 1.2 目的

神戸勤労者山岳会（KCAC）の会運営に必要な情報を一元管理し、会員・リーダー・事務局の業務効率化と情報共有の円滑化を実現する。

### 1.3 背景と課題

| 課題カテゴリ | 現状の問題 | 影響 |
|-------------|-----------|------|
| **フォーマットの分散** | 計画書がWord/Excel混在、スマホで作成困難 | 作成負荷が高い、形式が統一されない |
| **転記作業の多さ** | 会員情報を計画書に毎回手入力 | ミスが発生、時間がかかる |
| **情報の検索性不足** | 名簿・記録がファイルで分散管理 | 過去データの参照が困難 |
| **実績の見える化不足** | 例会数・参加者の集計ができない | 活動報告書作成に手間 |
| **装備管理の曖昧さ** | 貸出状況がスプレッドシートで管理 | 返却漏れ、二重貸出のリスク |

### 1.4 ゴール

```
Word/Excel様式から脱却
     ↓
データベース化による一元管理
     ↓
情報共有の円滑化・記録の継承
```

## 2. ユーザー別メリット

### 2.1 一般会員

| 従来 | システム導入後 | メリット |
|------|---------------|---------|
| メールで例会案内を受け取り、返信で申込 | Webで例会一覧を確認、ワンクリック申込 | **手間削減・申込状況がリアルタイムでわかる** |
| 計画書の集合場所・時間を確認するのに書類を探す | Webで過去の例会詳細を検索 | **必要な情報にすぐアクセス** |
| 装備を借りるとき事務局に確認 | Webで空き状況を確認、貸出申請 | **自分で確認・手続き完結** |
| 自分の参加実績がわからない | マイページで参加履歴を確認 | **自己管理しやすい** |

### 2.2 CL（チーフリーダー）

| 従来 | システム導入後 | メリット |
|------|---------------|---------|
| 計画書をWord/Excelで作成、会員情報を転記 | フォームに入力、会員は選択するだけ | **転記ミスゼロ・作成時間短縮** |
| 参加者の保険口数を名簿で確認 | 自動で保険口数をチェック | **確認漏れ防止** |
| 記録をWordで作成、メールで提出 | 計画書から情報を引き継いで記録作成 | **二重入力不要** |
| 例会案内をGoogleフォームで作成 | 統一フォームで案内作成 | **フォーマット統一・入力が簡単** |

### 2.3 事務局・運営委員

| 従来 | システム導入後 | メリット |
|------|---------------|---------|
| 例会企画を部署別にExcelで集約 | カレンダーで一元管理、重複チェック | **日程調整が容易** |
| 会員名簿をスプレッドシートで管理 | データベースで管理、検索・更新が容易 | **最新情報を維持しやすい** |
| 装備貸出をスプレッドシートで管理 | 貸出・返却をシステムで管理、期限超過を自動通知 | **管理漏れ防止** |
| 例会数・参加者を手作業で集計 | ダッシュボードで自動集計 | **報告資料作成が簡単** |
