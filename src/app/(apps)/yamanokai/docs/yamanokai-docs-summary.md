## 1. 企画書 - KCACシステム化計画

**作成日**: 2025年9月11日（機関誌部 逢坂）
**修正日**: 2025年12月7日

### 1.1 現状と問題点

#### 計画書と記録に関する問題

| 問題               | 詳細                                                               |
| ------------------ | ------------------------------------------------------------------ |
| フォーマットの問題 | 計画書がExcelまたはWordで作成されている                            |
| デバイス対応       | PCを持たない会員がスマホで計画書を作成している（できない人もいる） |
| 転記作業           | 計画書への会員情報の転記作業が多い                                 |
| 記録保存           | 山行記録方法が機関誌「すずの子」掲載だけで記録保存できていない     |

#### 会運営に関する問題

| 問題           | 詳細                                                             |
| -------------- | ---------------------------------------------------------------- |
| 情報分散       | 名簿関連書類が複数存在し、最新情報の更新ができていないものもある |
| 検索性・共有性 | 例会数・参加者の集計ができない（記録を見て数えている）           |
| 参加実績       | 会員の例会参加実績がわからない                                   |

### 1.2 システム化の目的

1. Word様式から脱却し、**データベース化を前提としたシステム**を設計
2. **会員情報、山行記録、装備貸出・返却、会行事・講座を一元管理**
3. **情報共有を円滑化**
4. **活動記録を長期的に保存**し、後世へ継承

### 1.3 対象範囲

| 管理情報                       | 現在の作成方法                               |
| ------------------------------ | -------------------------------------------- |
| 例会企画                       | 企画をメールで集めてエクセルでまとめる       |
| 例会案内                       | グーグルフォームにてCLが入力                 |
| 計画書                         | CLがエクセルまたはワードで作成               |
| 記録                           | 記録担当者がワードで作成し編集担当者へメール |
| CL用会員名簿                   | 事務局長がスプレッドシートにて管理           |
| クリーンハイク名簿             | 事務局長がエクセルで作成                     |
| 岩例会参加者名簿               | 山行部がエクセルで作成                       |
| 岩講習会実施記録               | 山行部がエクセルで作成                       |
| 新入会員さんようこそ           | 新入会員へワード原稿をメールで依頼           |
| 会員台帳                       | 事務局長がエクセルで作成                     |
| テント・無線機・ロープ使用管理 | スプレッドシートに使用者が登録               |

### 1.4 システム要件定義

| 項目         | 内容                                                                 |
| ------------ | -------------------------------------------------------------------- |
| システム概要 | 山岳会員の情報を一元管理し、検索・更新・分析を容易にする             |
| 対象ユーザー | 一般会員、事務局、運営委員、三役                                     |
| 利用環境     | Webブラウザ（PC・スマホ対応）、クラウドサーバ                        |
| 機能要件     | 会員情報管理、検索・フィルタ、データ出力、セキュリティ、バックアップ |
| 非機能要件   | 操作性、セキュリティ対応                                             |
| 運用・保守   | 定期メンテナンス、問い合わせ対応フロー                               |

### 1.5 システム概要図

```
┌─────────────┐     ┌─────────────────────┐     ┌─────────────────────┐

│ ・入会年月  │     │                     │     │ ・カテゴリ管理      │
│ ・講座受講歴│     └─────────┬───────────┘     └─────────────────────┘
└──────┬──────┘               │
         │                       │
         ▼                       ▼
┌─────────────┐     ┌─────────────────────┐
│クリーンハイク│     │      計画書        │
│   名簿      │     │      例会記録      │

                              │
                              ▼
```

**ポイント**:

- 会情報を一元管理
- データベースより必要情報を出力するシステムを構築する（**外注する**）
- 会員全員が同じところにアクセスして共有する（各専門部のスケジュール重複防止）

## まとめ

### ドキュメント間の関連

```
①例会企画 ─────▶ ②例会案内 ─────▶ ③計画書 ─────▶ ⑤例会記録
 (年間計画)      (参加募集)      (実施前)      (実施後)
                                    │
                                    ▼
                              ④装備表
                            (持ち物確認)
```

### システム化による改善ポイント

1. **企画から記録までの一貫管理**
   - 例会企画 → 例会案内 → 計画書 → 例会記録の流れをシステムで管理

2. **会員情報の一元化**
   - 計画書への会員情報自動入力
   - 参加実績の自動集計

3. **検索・集計機能**
   - 例会数・参加者の自動集計
   - 会員ごとの参加履歴表示

4. **マルチデバイス対応**
   - スマホからの計画書作成・閲覧
   - PC不要で参加申込可能

5. **記録の長期保存**
   - データベースによる活動記録の永続化
   - 機関誌だけでなくシステムでの記録保存

6. **装備管理の効率化**
   - 装備の貸出・返却をシステムで一元管理
   - カテゴリ別（テント/ロープ/無線機/登攀具/調理器具）の整理
   - 返却期限超過の自動警告表示
   - 例会と紐づけた装備貸出の記録
   - 会員ごとの貸出履歴の可視化
   - 装備状態（良好/要点検/修理中）の管理
