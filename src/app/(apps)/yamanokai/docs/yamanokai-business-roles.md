# 山の会（KCAC）業務フローと権限設計

---

## 1. 概要

このドキュメントは、山の会（KCAC）システムの業務フローと必要な権限を整理したものです。

---

## 2. 組織構成

### 2.1 役職・ロール

| ロール | 説明 | 主な責任 |
|--------|------|----------|
| **三役** | 会長・副会長・事務局長 | 会全体の運営、緊急時の連絡先 |
| **事務局** | 事務局長を中心とした運営チーム | 会員管理、名簿管理、総会運営 |
| **運営委員** | 各専門部の代表者等 | 例会企画の調整、会の意思決定 |
| **専門部長** | 各専門部のリーダー | 部の活動計画、例会の承認 |
| **CL（チーフリーダー）** | 例会の責任者 | 例会案内作成、計画書作成、当日の統括 |
| **SL（サブリーダー）** | CLの補佐役 | CLの補佐、参加者サポート |
| **一般会員** | 会に所属する会員 | 例会への参加、装備の借用 |
| **記録担当** | 例会記録の作成者 | 山行記録の作成、提出 |
| **機関誌部** | 機関誌「すずの子」の編集 | 記録の編集、機関誌発行 |

### 2.2 専門部一覧

| 部署名 | 主な活動 | 担当する例会の例 |
|--------|----------|-----------------|
| **ハイキング部** | 初心者向けハイキング | お試しハイク、近郊の山、縦走トレーニング |
| **山行部** | 本格的な登山・岩登り・沢登り | アルパイン、岩登りA/B/C、沢登り |
| **教育部** | 講座・教室の運営 | 夏山登山教室、雪山講座、ステップアップ講座 |
| **自然保護部** | 環境保全活動 | クリーンハイク、森守りボランティア |
| **組織部** | 会行事の運営 | 総会、納山祭、新人歓迎会 |

## 3. 業務フロー

### 3.1 例会の流れ

```
①例会企画 → ②例会案内 → ③計画書作成 → ④山行実施 → ⑤例会記録
 (年間計画)   (参加募集)    (実施準備)      (当日)      (報告)
     │            │            │              │           │
  運営委員      CL         CL          CL・参加者    記録担当
  専門部長                                           機関誌部
```

### 3.2 各業務の詳細

#### ① 例会企画（年間計画）

| 項目 | 内容 |
|------|------|
| **担当者** | 運営委員、専門部長 |
| **作業内容** | 年間・四半期の例会スケジュールを策定 |
| **入力情報** | 日付、担当部、タイトル、CL候補 |
| **成果物** | 例会企画カレンダー |
| **注意点** | 部署間の日程重複を避ける |

#### ② 例会案内（参加募集）

| 項目 | 内容 |
|------|------|
| **担当者** | CL |
| **作業内容** | 参加者を募集する案内を作成 |
| **入力情報** | 山名、日程、グレード、集合場所、コース、申込期限、必要保険口数 |
| **成果物** | 例会案内 |
| **注意点** | 参加条件（保険口数、講座受講歴）を明記 |

#### ③ 計画書作成（実施準備）

| 項目 | 内容 |
|------|------|
| **担当者** | CL |
| **作業内容** | 山行計画書の作成、参加者情報の収集 |
| **入力情報** | 行程詳細、参加者（役割付き）、緊急連絡先、エスケープルート |
| **成果物** | 山行計画書、装備表 |
| **注意点** | 担当部長への提出、三役への連絡先共有 |

#### ④ 山行実施（当日）

| 項目 | 内容 |
|------|------|
| **担当者** | CL、SL、参加者 |
| **作業内容** | 装備・体調確認、山行、下山後ミーティング |
| **注意点** | 最終下山報告、中止時の担当部長への連絡 |

#### ⑤ 例会記録（報告）

| 項目 | 内容 |
|------|------|
| **担当者** | 記録担当 → 機関誌部（編集） |
| **作業内容** | 山行記録の作成、機関誌への掲載 |
| **入力情報** | コースタイム、本文、コース状況、特記事項（ヒヤリハット） |
| **成果物** | 例会記録（機関誌「すずの子」掲載） |

## 4. 権限設計

### 4.1 権限マトリクス

| 機能 | 一般会員 | CL | 記録担当 | 専門部長 | 事務局 | 三役 |
|------|:-------:|:--:|:-------:|:-------:|:------:|:----:|
| **ダッシュボード閲覧** | ○ | ○ | ○ | ○ | ○ | ○ |
| **例会企画 閲覧** | ○ | ○ | ○ | ○ | ○ | ○ |
| **例会企画 作成・編集** | - | - | - | ○ | ○ | ○ |
| **例会案内 閲覧** | ○ | ○ | ○ | ○ | ○ | ○ |
| **例会案内 作成・編集** | - | ○※1 | - | ○ | ○ | ○ |
| **例会 申込** | ○ | ○ | ○ | ○ | ○ | ○ |
| **計画書 閲覧** | ○ | ○ | ○ | ○ | ○ | ○ |
| **計画書 作成・編集** | - | ○※1 | - | ○ | ○ | ○ |
| **計画書 承認** | - | - | - | ○ | ○ | ○ |
| **例会記録 閲覧** | ○ | ○ | ○ | ○ | ○ | ○ |
| **例会記録 作成・編集** | - | ○※1 | ○※1 | ○ | ○ | ○ |
| **例会記録 掲載承認** | - | - | - | - | ○ | ○ |
| **会員情報 閲覧（基本）** | ○ | ○ | ○ | ○ | ○ | ○ |
| **会員情報 閲覧（詳細）** | △※2 | ○※3 | ○※3 | ○ | ○ | ○ |
| **会員情報 作成・編集** | - | - | - | - | ○ | ○ |
| **装備 閲覧** | ○ | ○ | ○ | ○ | ○ | ○ |

| **装備 登録・編集** | - | - | - | - | ○ | ○ |
| **統計・レポート** | △※4 | △※4 | △※4 | ○ | ○ | ○ |

**凡例**
- ○: 可能
- △: 一部可能
- -: 不可

**注釈**
- ※1: 自分が担当する例会のみ
- ※2: 自分の情報のみ閲覧可能
- ※3: 担当例会の参加者情報のみ閲覧可能
- ※4: 自分の参加実績のみ閲覧可能

### 4.2 ロール別の主要権限

#### 一般会員

```
できること:
├── 例会案内の閲覧・申込
├── 自分の参加履歴確認
├── 装備の貸出・返却
└── 自分の会員情報確認

できないこと:
├── 例会案内・計画書・記録の作成
├── 他会員の詳細情報閲覧
└── 会員・装備の登録
```

#### CL（チーフリーダー）

```
できること:
├── 例会案内の作成・編集（自分担当分）
├── 計画書の作成・編集（自分担当分）
├── 例会記録の作成・編集（自分担当分）
├── 参加者の保険口数・講座受講歴確認
├── 参加申込者の一覧確認
└── 装備の貸出管理

できないこと:
├── 他CLの例会案内・計画書の編集
├── 会員情報の登録・編集
└── 例会企画の作成
```

#### 事務局

```
できること:
├── 全機能へのフルアクセス
├── 会員情報の登録・編集・削除
├── 装備の登録・編集・削除
├── 例会企画の作成・編集
├── 統計・レポートの閲覧
└── 計画書の承認
```

### 4.3 データアクセス制御

| データ種別 | 一般会員 | CL | 事務局 |
|-----------|---------|-----|--------|
| **会員 - 氏名** | ○ | ○ | ○ |
| **会員 - 電話番号** | - | ○※ | ○ |
| **会員 - メールアドレス** | - | ○※ | ○ |
| **会員 - 住所** | - | - | ○ |
| **会員 - 緊急連絡先** | - | ○※ | ○ |
| **会員 - 保険口数** | 自分のみ | ○※ | ○ |
| **会員 - 持病** | 自分のみ | ○※ | ○ |
| **会員 - 生年月日** | - | ○※ | ○ |

※ CLが担当する例会の参加者に限定

## 5. 承認フロー

### 5.1 計画書の承認フロー

```
CL作成 → 担当部長確認 → 事務局承認 → 山行実施 → 完了
  │           │            │           │         │
[下書き]   [提出済]     [承認済]    [実施中]   [完了]
```

### 5.2 例会記録の承認フロー

```
記録担当作成 → CL確認 → 機関誌部編集 → 掲載
     │           │           │          │
  [下書き]    [提出済]    [編集中]   [掲載済]
```

## 6. 例会参加の条件

### 6.1 保険口数による参加制限

| 例会種類 | 必要口数 | 例 |
|----------|---------|-----|
| ハイキング | 3口以上 | 近郊の山、お試しハイク |
| 岩登りA・沢入門 | 4口以上 | 岩登りA（初心者向け） |
| アルパイン・雪山・岩BC・沢 | 8口以上 | 西穂高岳、雪山ハイキング |

### 6.2 講座受講による参加制限

| 例会種類 | 必要な受講歴 |
|----------|-------------|
| 岩登りA以上 | 岩座学 |
| 初級雪山ハイキング | 雪山ハイキング講座 |
| 中級雪山ハイキング | 初級雪山ハイキング講座 + 中級雪山アイゼントレ |

## 7. システムで管理すべき権限チェック

### 7.1 例会申込時の自動チェック

```typescript
// 申込時のチェック項目
interface EventApplicationCheck {
  // 保険口数チェック
  hasRequiredInsurance: boolean;  // 会員の口数 >= 必要口数

  // 講座受講チェック
  hasRequiredCourses: boolean;    // 必要な講座を受講済みか

  // 申込期限チェック
  isWithinDeadline: boolean;      // 申込期限内か

  // 定員チェック
  hasCapacity: boolean;           // 定員に空きがあるか
}
```

### 7.2 計画書作成時の自動入力

```typescript
// 参加者選択時に自動入力される項目
interface ParticipantAutoFill {
  name: string;           // 氏名
  birthday: Date;         // 生年月日
  gender: string;         // 性別
  insurance: number;      // 保険口数
  bloodType: string;      // 血液型
  address: string;        // 住所
  phone: string;          // 電話番号
  emergencyContact: {     // 緊急連絡先
    name: string;
    phone: string;
    relationship: string;
  };
  medicalHistory: string; // 持病
  cocoHeliId: string;     // ココヘリID
}
```
