## 1. 運行入力

### 概要

ドライバーが自分の運行実績を入力するページです。配車された便に対して、出発時刻・到着時刻・走行距離・高速代などを入力します。

### アクセス方法

| 方法           | 詳細                                                   |
| -------------- | ------------------------------------------------------ |
| URL            | `/tbm/driver/driveInput`                               |
| ナビゲーション | 画面上部メニュー → **「ドライバーメニュー」** → **「運行入力」** |
| 対象ユーザー   | ログイン済みユーザー（ドライバー）                     |

> 💡 ナビゲーションには「🚚 運行入力」アイコン付きで表示されます

### 機能一覧

#### 1.1 日付選択

画面上部の日付切り替えボタンで入力対象日を変更できます。デフォルトは当日。

#### 1.2 配車一覧表示

当日に配車されている便の一覧が表示されます。

**表示項目**

| 項目         | 説明                         |
| ------------ | ---------------------------- |
| 便名         | 配車された便の名称           |
| 路線名       | 便の路線名                   |
| 出発時刻     | 予定出発時刻                 |
| 到着時刻     | 予定到着時刻                 |
| 車両         | 割り当てられた車両           |
| ステータス   | 運行/締め/承認の状態         |

#### 1.3 運行データ入力

各便のカードをクリックすると、運行データの入力フォームが開きます。

**入力項目**

| 項目               | 入力形式 | 必須 | 説明                         |
| ------------------ | -------- | ---- | ---------------------------- |
| 出発オドメーター   | 数値     | -    | 出発時のオドメーター値       |
| 到着オドメーター   | 数値     | -    | 到着時のオドメーター値       |
| 郵便高速代         | 数値     | -    | 郵便向け有料利用料           |
| 一般高速代         | 数値     | -    | 一般向け有料利用料           |
| 給油量             | 数値     | -    | 給油量（リットル）           |
| 備考               | テキスト | -    | 特記事項                     |

#### 1.4 ステータス変更

| ステータス | 操作           | 説明                             |
| ---------- | -------------- | -------------------------------- |
| 運行       | ドライバー操作 | 運行データを入力完了             |
| 締め       | ドライバー操作 | データを確定（修正不可にする）   |
| 承認       | 管理者操作     | 管理者が内容を承認               |

> ⚠️ 「締め」後は原則として修正不可。管理者のみ承認取り消し可能。

## 2. 月間予定

### 概要

ドライバーの月間スケジュールをカレンダー形式で確認するページです。

### アクセス方法

| 方法           | 詳細                                                   |
| -------------- | ------------------------------------------------------ |
| URL            | `/tbm/driver/monthly-schedule`                         |
| ナビゲーション | 画面上部メニュー → **「ドライバーメニュー」** → **「月間予定」** |
| 対象ユーザー   | ログイン済みユーザー（ドライバー）                     |

> 💡 ナビゲーションには「📅 月間予定」アイコン付きで表示されます

### 機能一覧

#### 2.1 月切り替え

- 「◀」「▶」ボタンで前月/翌月に移動
- 月入力フィールドで直接年月を選択

#### 2.2 カレンダー表示

週単位のカレンダー形式で月間スケジュールを表示します。

**セル表示内容**

| 項目     | 説明                       |
| -------- | -------------------------- |
| 日付     | 日付と曜日                 |
| 便名     | 配車された便の名称         |
| 車両     | 割り当てられた車両ナンバー |
| 出発時刻 | 予定出発時刻               |

**日付セルの色分け**

| 色     | 意味       |
| ------ | ---------- |
| 黄色   | 当日       |
| 淡い赤 | 土日・祝日 |
| 白     | 平日       |
| グレー | 当月以外   |

#### 2.3 運行入力画面へのリンク

日付をクリックすると、その日の運行入力画面を別タブで開きます。

## 設計者目線の詳細仕様

### コンポーネント構成

```
driver/
├── driveInput/
│   ├── page.tsx                # サーバーコンポーネント
│   ├── DriveInputCC.tsx        # クライアントコンポーネント
│   ├── DriveScheduleItem.tsx   # 配車アイテム
│   ├── Updator.tsx             # 更新ロジック
│   ├── driveInput-page-type.ts # 型定義
│   └── FormList/               # 入力フォーム部品
│       ├── Base.tsx
│       ├── formList.tsx
│       ├── Fuel.tsx
│       └── GoBack.tsx
├── monthly-schedule/
│   └── page.tsx               # カレンダー表示
└── kintai/
    ├── page.tsx               # 出退勤管理
    └── components/
        └── InlineEditField.tsx
```

### データフロー（運行入力）

```
page.tsx (RSC)
    ↓ [dateSwitcherTemplate で日付パラメータ取得]
    ↓ [getDriveInputPageData で配車データ取得]
DriveInputCC.tsx (RCC)
    ↓ [配車一覧表示]
DriveScheduleItem.tsx
    ↓ [各配車の詳細・入力フォーム]
Updator.tsx
    ↓ [データ更新処理]
```

### URLクエリパラメータ

#### 運行入力

| パラメータ         | 型     | デフォルト | 説明                 |
| ------------------ | ------ | ---------- | -------------------- |
| from               | Date   | 当日       | 表示対象日           |
| tbmDriveInputUserId | number | -          | 対象ドライバーID     |

#### 月間予定

| パラメータ | 型     | デフォルト | 説明           |
| ---------- | ------ | ---------- | -------------- |
| yearMonth  | string | 当月       | 表示年月(YYYY-MM) |
| driverId   | number | -          | 対象ドライバーID |

### ユーザー特定ロジック

```typescript
// スコープからドライバーIDを取得
const { tbmDriveInputUserId } = scopes.getTbmScopes()
const user = await prisma.user.findUnique({ where: { id: tbmDriveInputUserId } })
```

## 機能間の関連性

| 関連機能     | 関連内容                                       |
| ------------ | ---------------------------------------------- |
| 配車設定     | 配車データを運行入力の対象とする               |
| 運行明細     | 入力した運行データが運行明細に反映             |
| 燃費管理     | 給油データが燃費計算に使用される               |
| 累積距離記帳 | オドメーターデータが距離計算に使用される       |
| 給与算定     | 運行データが給与計算に使用される               |

- [ ] ドライバーにログインアカウントが発行されているか
