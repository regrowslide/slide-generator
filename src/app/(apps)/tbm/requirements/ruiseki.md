## 1. ユーザー目線の機能一覧

### 概要

累積距離記帳ページでは、ドライバーごとに使用した車両の走行距離・燃費・燃料費を確認できます。給与算定の基礎データとして使用されます。

### アクセス方法

| 方法           | 詳細                                                       |
| -------------- | ---------------------------------------------------------- |
| URL            | `/tbm/ruiseki`                                              |
| ナビゲーション | 画面上部メニュー → **「レポート①」** → **「累積距離記帳」** |
| 対象ユーザー   | 事務員（isJimuin）、所長（isShocho）、システム管理者       |

> 💡 ナビゲーションには「🧮 累積距離記帳」アイコン付きで表示されます

---

### 機能一覧

#### 1.1 月切り替え

画面上部の日付切り替えボタンで表示月を変更できます。

#### 1.2 ドライバー別車両使用実績

各ドライバーのカードに、当月使用した車両ごとの実績を表示します。

| 列名       | 説明                         |
| ---------- | ---------------------------- |
| 車番       | 使用した車両のナンバー       |
| 走行距離計 | 当月の走行距離合計（km）     |
| 平均燃費   | 平均燃費（km/L）             |
| 総給油量   | 当月の給油量合計（L）        |
| 使用金額   | 燃料費合計（円）             |

> ⚠️ 使用した車両がないドライバーは「データがありません」と表示されます

---

### 機能間の関連性

| 関連機能     | 関連内容                                       |
| ------------ | ---------------------------------------------- |
| 配車設定     | ドライバーと車両の紐付けデータを参照           |
| 燃費管理     | 車両の燃費データを共有                         |
| 営業所設定   | 軽油単価設定を参照                             |
| 給与算定     | 累積距離データを給与計算に使用                 |

## 2. 設計者目線の詳細仕様

### 2.1 コンポーネント構成

```
ruiseki/
└── page.tsx    # サーバーコンポーネント（ページエントリ + 表示）
```

### 2.2 データフロー

```
page.tsx (RSC)
    ↓ [dateSwitcherTemplate で日付パラメータ取得]
    ↓ [getTbmBase_MonthConfig で軽油単価取得]
    ↓ [fetchRuisekiKyoriKichoData でドライバー別データ取得]
    ↓   └─ getNenpiDataByCar で車両別燃費データ取得
    ↓   └─ TbmDriveSchedule でドライバー×車両の紐付け取得
    ↓ [R_Stack でドライバーカード一覧表示]
    ↓ [CsvTable で車両使用実績一覧表示]
```

### 2.3 URLクエリパラメータ

| パラメータ | 型   | デフォルト | 説明       |
| ---------- | ---- | ---------- | ---------- |
| from       | Date | 月初       | 表示開始日 |
| to         | Date | 月末       | 表示終了日 |

### 2.4 データ集計ロジック

```typescript
// ドライバーが使用した車両を抽出
const allCars = TbmDriveSchedule.reduce((acc, cur) => {
  const { TbmVehicle } = cur
  if (!acc.find(v => v.id === TbmVehicle?.id)) {
    acc.push(TbmVehicle)
  }
  return acc
}, [])

// 各車両の燃費データをマージ
allCars = allCars.map(car => {
  const fuelData = nenpiKanriDataListByCar.find(v => v?.vehicle?.id === car.id)
  return { car, ...fuelData }
})
```

### 2.5 燃費管理との違い

| 項目       | 燃費管理         | 累積距離記帳         |
| ---------- | ---------------- | -------------------- |
| 軸         | 車両             | ドライバー           |
| 対象車両   | 営業所の全車両   | ドライバーが使用した車両のみ |
| 用途       | 車両管理         | 給与算定             |

## 3. テーブル定義書

### 3.1 使用テーブル

累積距離記帳では主に以下のテーブルを参照します：

| テーブル名        | 用途                           |
| ----------------- | ------------------------------ |
| User              | ドライバー情報                 |
| TbmDriveSchedule  | ドライバー×車両の紐付け        |
| TbmVehicle        | 車両情報                       |
| TbmRefuelHistory  | 給油履歴（燃費計算用）         |
| TbmBase_MonthConfig | 軽油単価                     |

詳細なテーブル定義は [燃費管理](./nempiKanri.md#3-テーブル定義書) を参照してください。

## 4. チェックリスト

### 導入時の確認事項

- [ ] 配車設定でドライバーと車両が紐付けされているか
- [ ] 車両の給油履歴が登録されているか
- [ ] 営業所設定で軽油単価が設定されているか

### 運用時の注意事項

- [ ] 承認済みの配車データのみが集計対象
- [ ] 未配車（車両未設定）のデータは含まれない
- [ ] 燃費データは燃費管理ページと同じロジックで計算
