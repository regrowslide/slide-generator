## 1. ユーザー目線の機能一覧

### 概要

ETC連携ページでは、ETCカードの利用明細データ（CSV形式）をインポートし、運行データと紐付けて管理します。

### アクセス方法

| 方法           | 詳細                                                   |
| -------------- | ------------------------------------------------------ |
| URL            | `/tbm/etc`                                             |
| ナビゲーション | 画面上部メニュー → **「営業所メニュー」** → **「ETC連携」** |
| 対象ユーザー   | 事務員（isJimuin）、所長（isShocho）、システム管理者   |

---

### 機能一覧

#### 1.1 車両選択

ETC明細を表示・インポートする対象車両を選択します。

| 操作     | 説明                                     |
| -------- | ---------------------------------------- |
| 車両選択 | ドロップダウンから車両を選択             |
| 検索     | 選択した車両のETCデータを表示            |

> ⚠️ 車両を選択しないとデータインポート・確認機能は使用できません

#### 1.2 データインポートタブ

ETCカード会社からダウンロードしたCSVファイルをインポートします。

| 機能             | 説明                                         |
| ---------------- | -------------------------------------------- |
| CSVファイル選択  | Shift-JIS形式のCSVファイルを選択             |
| アップロードプレビュー | 読み込んだデータの内容を確認               |
| インポート実施   | データベースに保存                           |

**CSVファイルの列構造**

| 列番号 | 内容             | 例              |
| ------ | ---------------- | --------------- |
| 1      | 利用開始日       | 25/08/01        |
| 2      | 利用開始時刻     | 09:30           |
| 3      | 利用終了日       | 25/08/01        |
| 4      | 利用終了時刻     | 10:15           |
| 5      | 利用IC（入口）   | 名古屋IC        |
| 6      | 利用IC（出口）   | 岡崎IC          |
| 7      | 元の料金         | 1,200           |
| 8      | 割引額           | 360             |
| 9      | 通行料金         | 840             |
| 10     | 車種             | 中型車          |
| 11     | 車両番号         | 名古屋100あ1234 |
| 12     | ETCカード番号    | 1234-5678-XXXX  |
| 13     | 備考             | （任意）        |

#### 1.3 データ確認とグルーピングタブ

インポートしたデータの確認・編集を行います。

| 機能                   | 説明                                               |
| ---------------------- | -------------------------------------------------- |
| 月選択                 | 表示する月を変更                                   |
| レコードの選択         | チェックボックスまたは行クリックで複数選択         |
| グループ化             | 選択したレコードを1つの運行としてまとめる         |
| グループ解除           | グループ化されたレコードを個別に戻す               |
| 運行データ紐付け       | グループ化したETCデータを配車スケジュールに紐付け |
| 月次データ一括削除     | 選択中の月のデータをすべて削除                     |

**グループ化の用途**

往復や複数区間を経由する運行では、複数のETC明細が発生します。これらを1つのグループにまとめることで、運行単位での高速代管理が可能になります。

**表示内容**

| 列           | 説明                                     |
| ------------ | ---------------------------------------- |
| 連番         | 表示順の番号                             |
| グループ     | グループ化操作・解除ボタン               |
| 利用年月日（自） | 利用開始日                           |
| 時分（自）   | 利用開始時刻                             |
| 利用年月日（至） | 利用終了日                           |
| 時分（至）   | 利用終了時刻                             |
| 利用IC（自） | 入口IC名                                 |
| 利用IC（至） | 出口IC名                                 |
| 通行料金     | 個別レコードの料金                       |
| グループ料金 | グループ合計金額（グループヘッダーのみ） |
| 紐付け運行   | 紐付けた運行データの情報                 |

**グループの色分け**

- 奇数グループ: 青系の背景色
- 偶数グループ: 緑系の背景色
- ヘッダー行: 濃い色
- 明細行: 薄い色

### 機能間の関連性

| 関連機能     | 関連内容                                               |
| ------------ | ------------------------------------------------------ |
| 営業所設定   | 車両マスタでETCカード番号を管理                        |
| 配車設定     | ETCデータを配車スケジュールに紐付け                    |
| 給与算定     | 紐付けたETC料金を高速代として給与計算に反映            |
| 車両詳細     | 車両ごとのETC利用明細を確認可能                        |

## 2. 設計者目線の詳細仕様

### 2.1 コンポーネント構成

```
etc/
├── page.tsx                    # メインページ（クライアントコンポーネント）
├── types.ts                    # 型定義
├── components/
│   ├── EtcImportForm.tsx       # インポートフォーム
│   ├── EtcFileUpload.tsx       # ファイルアップロード
│   ├── EtcDataTable.tsx        # データテーブル
│   └── EtcScheduleLinkModal.tsx # 運行紐付けモーダル
└── hooks/
    ├── useEtcData.tsx          # データ管理Hook
    ├── useEtcGrouping.tsx      # グループ化Hook
    └── useEtcSelection.tsx     # 行選択Hook
```

### 2.2 データフロー

```
[CSVファイル]
    ↓ EtcFileUpload でファイル読み込み
    ↓ Shift-JIS → UTF-8 変換
[useEtcData.importCsvData]
    ↓ CSVパース
    ↓ doTransaction でDBに一括保存
[EtcCsvRaw テーブル]
    ↓ useEtcData でデータ取得
    ↓ useEtcGrouping でグループ化
[TbmEtcMeisai テーブル] ← グループ情報
    ↓ EtcScheduleLinkModal で紐付け
[TbmDriveSchedule テーブル] ← 運行データ
```

### 2.3 URLクエリパラメータ

| パラメータ    | 型     | デフォルト | 説明           |
| ------------- | ------ | ---------- | -------------- |
| tbmVehicleId  | number | -          | 対象車両ID     |
| month         | Date   | 当月       | 表示対象月     |

### 2.4 CSVパース処理

```typescript
// 日付パース例（25/08/01 → 2025-08-01）
const parseJapaneseDate = (dateStr: string) => {
  const [year, month, day] = dateStr.split('/')
  return new Date(`20${year}-${month}-${day}`)
}

// 料金パース例（"1,200" → 1200）
const parseFee = (feeStr: string) => {
  const cleaned = feeStr.replace(/[^\d-]/g, '')
  return parseInt(cleaned, 10)
}
```

### 2.5 グループ化ロジック

1. ユーザーが複数レコードを選択
2. 「グループ化」ボタンクリック
3. `TbmEtcMeisai` レコードを新規作成
4. 選択した `EtcCsvRaw` レコードの `tbmEtcMeisaiId` を更新
5. `isGrouped: true` に変更

### 2.6 運行紐付けロジック

1. グループヘッダーの「紐付け」ボタンクリック
2. モーダルで対象日付の配車スケジュールを選択
3. `TbmEtcMeisai.tbmDriveScheduleId` を更新

### 2.7 ユニーク制約

`EtcCsvRaw` テーブルでは以下の組み合わせでユニーク:
- `tbmVehicleId`
- `fromDate`
- `fromTime`

→ 同一車両・同一日時の重複インポートを防止（upsertで処理）

## 3. テーブル定義書

### 3.1 EtcCsvRaw（ETC生データ）

| カラム名       | 型       | 必須 | 説明                       |
| -------------- | -------- | ---- | -------------------------- |
| id             | Int      | ○    | 主キー（自動採番）         |
| tbmVehicleId   | Int      | ○    | 車両ID（FK）               |
| fromDate       | DateTime | ○    | 利用開始日                 |
| fromTime       | String   | ○    | 利用開始時刻               |
| toDate         | DateTime | ○    | 利用終了日                 |
| toTime         | String   | -    | 利用終了時刻               |
| fromIc         | String   | -    | 入口IC                     |
| toIc           | String   | -    | 出口IC                     |
| fee            | Int      | ○    | 通行料金                   |
| originalFee    | Int      | -    | 元の料金                   |
| discountAmount | Int      | -    | 割引額                     |
| isGrouped      | Boolean  | ○    | グループ化フラグ           |
| tbmEtcMeisaiId | Int      | -    | 明細ID（FK、グループ化時） |
| cardNumber     | String   | -    | ETCカード番号              |
| carType        | String   | -    | 車種                       |
| remark         | String   | -    | 備考                       |
| createdAt      | DateTime | ○    | 作成日時                   |
| updatedAt      | DateTime | ○    | 更新日時                   |

#### ユニーク制約

- `unique_tbmVehicleId_fromDate_fromTime`: 同一車両・同一日時に1レコード

#### リレーション

- `TbmVehicle` → 1:N
- `TbmEtcMeisai` → N:1（グループ化時）

### 3.2 TbmEtcMeisai（ETC明細グループ）

| カラム名          | 型       | 必須 | 説明                   |
| ----------------- | -------- | ---- | ---------------------- |
| id                | Int      | ○    | 主キー（自動採番）     |
| tbmDriveScheduleId | Int     | -    | 配車スケジュールID（FK）|
| createdAt         | DateTime | ○    | 作成日時               |
| updatedAt         | DateTime | ○    | 更新日時               |

#### リレーション

- `EtcCsvRaw` → 1:N（グループ内のレコード）
- `TbmDriveSchedule` → N:1（紐付け先の運行）

## 4. チェックリスト

### 導入時の確認事項

- [ ] 車両マスタにETCカード番号が登録されているか
- [ ] CSVファイルの文字コードがShift-JISか確認
- [ ] CSVファイルの列構造が想定どおりか確認

### 運用時の注意事項

- [ ] 月次でETCデータをインポート
- [ ] 複数区間の運行はグループ化を実施
- [ ] グループ化後に運行データと紐付け
- [ ] 重複インポートはupsertで処理されるため問題なし

### よくあるトラブル

| 症状                     | 原因                           | 対処法                             |
| ------------------------ | ------------------------------ | ---------------------------------- |
| ファイルが読み込めない   | 文字コードの問題               | Shift-JIS形式で保存し直す          |
| データが表示されない     | 車両が選択されていない         | 車両を選択して検索ボタンを押す     |
| 日付がおかしい           | CSVの日付形式が異なる          | YY/MM/DD形式に修正する             |
| グループ化できない       | レコードが選択されていない     | チェックボックスで選択する         |
