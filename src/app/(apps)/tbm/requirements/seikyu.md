## 1. ユーザー目線の機能一覧

### 概要

請求書発行ページでは、取引先（荷主）ごとに月間の請求書と請求明細を作成・確認できます。

### アクセス方法

| 方法           | 詳細                                                     |
| -------------- | -------------------------------------------------------- |
| URL            | `/tbm/seikyu`                                            |
| ナビゲーション | 画面上部メニュー → **「レポート①」** → **「請求書発行」** |
| 対象ユーザー   | 事務員（isJimuin）、所長（isShocho）、システム管理者     |

> 💡 ナビゲーションには「📋 請求書発行」アイコン付きで表示されます

---

### 機能一覧

#### 1.1 月切り替え

画面上部の日付切り替えボタンで表示月を変更できます。

#### 1.2 顧客選択

ドロップダウンから請求対象の顧客（荷主）を選択します。

| 表示項目     | 説明                         |
| ------------ | ---------------------------- |
| 顧客名       | 荷主名                       |
| 取引件数     | 当月の承認済み運行スケジュール数 |

> ⚠️ 便設定で顧客が紐付けられていない場合は、顧客リストに表示されません

#### 1.3 請求書タブ

選択した顧客への請求書を表示します。

**請求書の構成**

| セクション       | 内容                                   |
| ---------------- | -------------------------------------- |
| 会社情報         | 自社名、住所、電話・FAX、振込先銀行    |
| 顧客情報         | 宛先顧客名、住所                       |
| 請求年月         | 対象年月                               |
| 便区分別集計     | 便区分ごとの回数・金額                 |
| 小計             | 税抜合計金額                           |
| 消費税           | 10%消費税                              |
| 合計             | 税込合計金額                           |

**便区分別詳細**

| 列名         | 説明                           |
| ------------ | ------------------------------ |
| 区分         | 便区分（郵便/一般等）          |
| 路線名       | 便の路線名                     |
| 便名         | 便の名称                       |
| 車種         | 使用車種                       |
| 回数         | 運行回数                       |
| 運賃単価     | 運賃の単価（変動がある場合は期間表示） |
| 運賃金額     | 運賃合計                       |
| 付帯単価     | 付帯作業の単価                 |
| 付帯料金     | 付帯作業合計                   |
| 通行料単価   | 通行料の単価                   |
| 通行料       | 通行料合計                     |

> 💡 運賃単価が月内で変更された場合は、期間別に単価が表示されます

#### 1.4 請求書用明細タブ

詳細な請求明細を表示します。路線ごとの内訳確認に使用します。

---

### 機能間の関連性

| 関連機能     | 関連内容                                       |
| ------------ | ---------------------------------------------- |
| 営業所設定   | 便の運賃・付帯作業料金、便と荷主の紐付けを参照 |
| 配車設定     | 承認済みの配車データを請求対象とする           |
| 運行明細     | 運行データの詳細確認                           |
| 共通設定     | 荷主マスタを参照                               |

## 2. 設計者目線の詳細仕様

### 2.1 コンポーネント構成

```
seikyu/
├── page.tsx                    # サーバーコンポーネント（ページエントリ）
├── (components)/
│   ├── CustomerSelector.tsx    # 顧客選択ドロップダウン
│   ├── InvoiceViewer.tsx       # 請求書表示
│   └── MeisaiViewer.tsx        # 明細表示
```

### 2.2 データフロー

```
page.tsx (RSC)
    ↓ [Mid_TbmRouteGroup_TbmCustomer から顧客一覧取得]
    ↓ [顧客ごとの取引件数を計算]
    ↓ [顧客が選択されている場合]
    ↓   └─ getInvoiceData で請求書データ取得
    ↓   └─ getMeisaiData で明細データ取得
    ↓ [BasicTabs で請求書/明細タブ表示]
```

### 2.3 URLクエリパラメータ

| パラメータ | 型     | デフォルト | 説明           |
| ---------- | ------ | ---------- | -------------- |
| from       | Date   | 月初       | 表示開始日     |
| to         | Date   | 月末       | 表示終了日     |
| customerId | number | -          | 選択中の顧客ID |

### 2.4 請求月判定ロジック（月末日跨ぎ対応）

月末最終日の深夜運行（翌日0時以降に終了）は、実際の運行日ではなく出発月で請求を計算します。

```typescript
// BillingHandler.getBillingMonth の処理
// 例: 1/31 23:00出発 → 2/1 01:00到着 の場合
//   → 1月分として請求

const billingMonth = BillingHandler.getBillingMonth(
  schedule.date,           // 運行日
  schedule.TbmRouteGroup.departureTime  // 出発時刻
)
```

### 2.5 運賃単価の変動対応

月内で運賃単価が変更された場合、`TbmRouteGroupFee.startDate` を基準に期間別の単価を計算します。

```typescript
// 運賃バリエーションの計算
const driverFeeVariations = calculatePriceVariations(feeInfos, 'driverFee', monthEnd)

// 結果例:
// [
//   { price: 10000, startDate: 2025-01-01, endDate: 2025-01-15 },
//   { price: 12000, startDate: 2025-01-16, endDate: 2025-01-31 }
// ]
```

### 2.6 手動編集機能

`TbmInvoiceManualEdit` テーブルに手動編集データが存在する場合、自動計算データを上書きします。

```typescript
// 手動編集データの取得
const manualEditData = await getInvoiceManualEdit({
  tbmCustomerId: customerId,
  yearMonth: whereQuery.gte,
})

// 手動編集がある場合は上書き
if (manualEditData.summaryByCategory) {
  finalSummaryByCategory = manualEditData.summaryByCategory
}
```

### 2.7 消費税計算

消費税率は10%で固定。小数点以下は切り捨て。

```typescript
const taxAmount = Math.floor(totalAmount * 0.1)
const grandTotal = totalAmount + taxAmount
```

## 3. テーブル定義書

### 3.1 Mid_TbmRouteGroup_TbmCustomer（便-荷主紐付け）

| カラム名        | 型   | 必須 | 説明           |
| --------------- | ---- | ---- | -------------- |
| id              | Int  | ○    | 主キー         |
| tbmRouteGroupId | Int  | ○    | 便ID（FK）     |
| tbmCustomerId   | Int  | ○    | 荷主ID（FK）   |

#### リレーション

- `TbmRouteGroup` → N:1
- `TbmCustomer` → N:1

### 3.2 TbmCustomer（荷主）

| カラム名 | 型     | 必須 | 説明               |
| -------- | ------ | ---- | ------------------ |
| id       | Int    | ○    | 主キー             |
| code     | String | -    | 荷主コード         |
| name     | String | ○    | 荷主名             |
| address  | String | -    | 住所               |

### 3.3 TbmRouteGroupFee（便運賃設定）

| カラム名        | 型       | 必須 | 説明               |
| --------------- | -------- | ---- | ------------------ |
| id              | Int      | ○    | 主キー             |
| tbmRouteGroupId | Int      | ○    | 便ID（FK）         |
| startDate       | DateTime | ○    | 適用開始日         |
| endDate         | DateTime | -    | 適用終了日         |
| driverFee       | Float    | -    | 運賃               |
| futaiFee        | Float    | -    | 付帯作業料金       |

### 3.4 TbmInvoiceManualEdit（請求書手動編集）

| カラム名        | 型       | 必須 | 説明               |
| --------------- | -------- | ---- | ------------------ |
| id              | Int      | ○    | 主キー             |
| tbmCustomerId   | Int      | ○    | 荷主ID（FK）       |
| yearMonth       | DateTime | ○    | 対象年月           |
| summaryByCategory | Json   | -    | 便区分別集計（JSON）|
| detailsByCategory | Json   | -    | 便区分別詳細（JSON）|

#### ユニーク制約

- `unique_tbmCustomerId_yearMonth`: 同一荷主・同一年月に1レコード

## 4. チェックリスト

### 導入時の確認事項

- [ ] 便設定で荷主が紐付けられているか
- [ ] 便の運賃・付帯作業料金が設定されているか
- [ ] 配車データが承認済みになっているか

### 運用時の注意事項

- [ ] 未承認の配車データは請求に含まれない
- [ ] 便に荷主が紐付けられていない運行は請求対象外
- [ ] 月末日跨ぎ運行は出発月で請求される
- [ ] 手動編集データがある場合は自動計算が上書きされる

### よくある確認ポイント

| 症状                     | 原因                           | 対処法                             |
| ------------------------ | ------------------------------ | ---------------------------------- |
| 顧客が表示されない       | 便に荷主が紐付けられていない   | 便設定で荷主を設定                 |
| 請求金額が0になる        | 運賃設定がない                 | 便詳細で運賃を設定                 |
| 回数が実際より少ない     | 配車データが未承認             | 配車設定で承認処理                 |
| 月末の運行が翌月になる   | 出発時刻の設定                 | 月末日跨ぎ判定を確認               |
