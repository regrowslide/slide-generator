## 1. ユーザー目線の機能一覧

### 概要

配車設定ページでは、運送便に対してドライバーと車両を割り当て、月間の配車スケジュールを管理します。

### アクセス方法

| 方法           | 詳細                                                     |
| -------------- | -------------------------------------------------------- |
| URL            | `/tbm/haisha`                                            |
| ナビゲーション | 画面上部メニュー → **「営業所メニュー」** → **「配車設定」** |
| 対象ユーザー   | 事務員（isJimuin）、所長（isShocho）、システム管理者     |

> 💡 ナビゲーションには「🚚 配車設定」アイコン付きで表示されます

---

### 機能一覧

#### 1.1 表示モード切り替え

2つの表示モードから選択できます。

| モード     | 説明                                           | 用途                                     |
| ---------- | ---------------------------------------------- | ---------------------------------------- |
| ドライバー | ドライバー × 日付のマトリクス表示              | ドライバーの月間勤務状況を確認する場合に |
| 便         | 便（ルート） × 日付のマトリクス表示            | 便の運行状況・担当者を確認する場合に     |

#### 1.2 並び順変更

| 並び順       | 説明                           |
| ------------ | ------------------------------ |
| 出発時間順   | 便の出発時刻が早い順（デフォルト） |
| 便コード順   | 便コードの昇順                 |
| 荷主コード順 | 荷主コードの昇順               |

#### 1.3 荷主フィルタ

特定の荷主の便だけを表示するフィルタ機能があります。

#### 1.4 配車の登録・編集

| 機能               | 説明                                                         |
| ------------------ | ------------------------------------------------------------ |
| 配車の新規登録     | カレンダーセルの「＋」ボタンをクリックして新規配車を登録     |
| 配車の編集         | 配車カード上の「✏️」ボタンをクリックして編集                 |
| 一括割り当て       | 便に対して複数日まとめてドライバー・車両を割り当て           |
| 運行入力画面へ遷移 | 配車カード上の「🚚」ボタンで運行入力画面を別タブで開く       |

#### 1.5 勤務状況の登録（ドライバーモード時）

ドライバーモードでは、各日の勤務状況を設定できます。

| 勤務状況   | 意味               |
| ---------- | ------------------ |
| 出勤       | 通常出勤           |
| 有給       | 有給休暇           |
| 欠勤       | 無届け欠勤         |
| 公休       | 所定休日           |
| 特別休暇   | 慶弔休暇等         |
| その他     | 上記以外の勤務状況 |

#### 1.6 承認機能

配車スケジュールには3段階のステータスがあります。

| ステータス | 説明                                         | 操作者     |
| ---------- | -------------------------------------------- | ---------- |
| 運行       | ドライバーが運行データを入力済み             | ドライバー |
| 締め       | ドライバーがデータを確定                     | ドライバー |
| 承認       | 管理者が内容を承認（事務員・所長が操作可能） | 管理者     |

> ⚠️ 「運行」「締め」ステータスはドライバー画面からのみ変更可能です

---

### 表示情報

#### 配車カードの表示内容

| 項目       | 説明                                   |
| ---------- | -------------------------------------- |
| 便名       | 便コードと便名                         |
| 時刻       | 出発時刻 - 到着時刻                    |
| 車両       | 割り当てられた車両（未設定時は赤字）   |
| ドライバー | 割り当てられたドライバー（未設定時は赤字） |
| 特記       | 備考欄の内容（15文字まで表示）         |
| ID         | 配車スケジュールID（右上に小さく表示） |

> ⚠️ 重複配車（同一ドライバーが同日に複数便）の場合、カードの背景が赤色で表示されます

---

### 機能間の関連性

| 関連機能       | 関連内容                                               |
| -------------- | ------------------------------------------------------ |
| 営業所設定     | 便マスタ、車両マスタ、ドライバーマスタのデータを参照   |
| 運行入力       | 配車データをベースにドライバーが運行実績を入力         |
| 運行明細       | 配車・運行データの一覧表示                             |
| 請求書発行     | 承認済みの配車データから請求金額を算出                 |
| 給与算定       | 配車データと運行実績から給与を計算                     |
| 出退勤管理     | 勤務状況データを参照                                   |

## 2. 設計者目線の詳細仕様

### 2.1 コンポーネント構成

```
haisha/
├── page.tsx                    # サーバーコンポーネント（ページエントリ）
├── components/
│   ├── HaishaTable.tsx         # メインテーブル（クライアントコンポーネント）
│   ├── HaishaTableContent.tsx  # テーブル本体
│   ├── HaishaTableSwitcher.tsx # モード切替UI
│   ├── TableRowBuilder.tsx     # 行構築ロジック
│   ├── CellComponents.tsx      # セル内コンポーネント群
│   ├── HaishaCard.tsx          # 配車カード
│   ├── DateThCell.tsx          # 日付ヘッダーセル
│   ├── PaginationControl.tsx   # ページネーション
│   ├── getListData.tsx         # データ取得ロジック
│   ├── UserTh.tsx              # ユーザーヘッダーセル
│   └── BulkAssignment/         # 一括割り当て関連
│       ├── BulkAssignmentModal.tsx
│       ├── MonthCalendarSelector.tsx
│       └── useBulkAssignmentModal.tsx
├── hooks/
│   ├── useHaishaData.tsx       # データ管理Hook
│   ├── usePagination.tsx       # ページネーションHook
│   └── useBulkAssignment.tsx   # 一括割り当てHook
├── server-actions/
│   └── bulkAssignSchedule.ts   # 一括割り当てサーバーアクション
└── types/
    └── haisha-page-types.ts    # 型定義
```

### 2.2 データフロー

```
page.tsx (RSC)
    ↓ [初期データ取得: tbmBase, 日付範囲]
HaishaTable.tsx (RCC)
    ↓ [useHaishaData Hook でデータ取得]
    ↓ [mode: ROUTE or DRIVER]
HaishaTableContent.tsx
    ↓ [TableRowBuilder で行構築]
CellComponents.tsx
    ↓ [各セルの表示・編集]
```

### 2.3 表示モード詳細

#### ドライバーモード（DRIVER）

- 縦軸: ドライバー一覧
- 横軸: 月の日付
- セル内容:
  - 勤務状況セレクター
  - 該当ドライバーの配車カード
  - 新規追加ボタン

#### 便モード（ROUTE）

- 縦軸: 便一覧
- 横軸: 月の日付
- セル内容:
  - 該当便の配車カード
  - 新規追加ボタン
  - 一括割り当てボタン（便行ヘッダー）

### 2.4 URL クエリパラメータ

| パラメータ    | 型       | デフォルト    | 説明                     |
| ------------- | -------- | ------------- | ------------------------ |
| mode          | string   | 'ROUTE'       | 表示モード               |
| from          | Date     | 月初          | 表示開始日               |
| to            | Date     | 月末          | 表示終了日               |
| sortBy        | string   | 'departureTime' | 並び順                 |
| tbmCustomerId | number   | -             | 荷主フィルタ             |

### 2.5 主要な型定義

```typescript
// 配車スケジュール（拡張型）
type TbmDriveScheduleWithDuplicated = TbmDriveSchedule & {
  TbmRouteGroup: TbmRouteGroup
  TbmVehicle: TbmVehicle & { OdometerInput: OdometerInput[] }
  User: { id: number; name: string }
  duplicated: boolean  // 重複配車フラグ
}

// 一括割り当てフォーム
interface BulkAssignmentFormData {
  userId: number | null
  tbmVehicleId: number | null
  selectedDates: Date[]
}
```

### 2.6 重複チェックロジック

配車データ取得時に、同一ドライバーが同日に複数の便に割り当てられている場合、`duplicated: true` フラグを付与。UI上で赤背景で表示して警告。

### 2.7 一括割り当て機能

1. 便の行ヘッダーから「一括割り当て」ボタンをクリック
2. カレンダーで対象日を複数選択
3. ドライバー・車両を選択（どちらか一方のみでも可）
4. 「一括割り当て実行」で選択した全日に配車レコードを作成

### 2.8 委託パターン（TbmRouteGroupCalendar）との連携

便モードでは、便の「委託パターン」（営業所設定で設定した年間稼働日）を参照して、カレンダーセルの背景色を変更。

## 3. テーブル定義書

### 3.1 TbmDriveSchedule（配車スケジュール）

| カラム名        | 型       | 必須 | 説明                         |
| --------------- | -------- | ---- | ---------------------------- |
| id              | Int      | ○    | 主キー（自動採番）           |
| date            | DateTime | ○    | 運行日                       |
| tbmBaseId       | Int      | ○    | 営業所ID（FK）               |
| tbmRouteGroupId | Int      | ○    | 便ID（FK）                   |
| userId          | Int      | -    | ドライバーID（FK）           |
| tbmVehicleId    | Int      | -    | 車両ID（FK）                 |
| remark          | String   | -    | 備考                         |
| driven          | Boolean  | -    | 運行フラグ                   |
| confirmed       | Boolean  | -    | 締めフラグ                   |
| approved        | Boolean  | -    | 承認フラグ                   |
| createdAt       | DateTime | ○    | 作成日時                     |
| updatedAt       | DateTime | ○    | 更新日時                     |

#### ユニーク制約

- `unique_tbmRouteGroupId_date`: 同一便・同一日に1レコードのみ

#### リレーション

- `TbmBase` → 1:N
- `TbmRouteGroup` → 1:N
- `User` → 1:N
- `TbmVehicle` → 1:N

### 3.2 UserWorkStatus（勤務状況）

| カラム名   | 型       | 必須 | 説明                   |
| ---------- | -------- | ---- | ---------------------- |
| id         | Int      | ○    | 主キー（自動採番）     |
| userId     | Int      | ○    | ユーザーID（FK）       |
| date       | DateTime | ○    | 対象日                 |
| workStatus | String   | ○    | 勤務状況コード         |
| createdAt  | DateTime | ○    | 作成日時               |
| updatedAt  | DateTime | ○    | 更新日時               |

#### ユニーク制約

- `unique_userId_date`: 同一ユーザー・同一日に1レコードのみ

#### 勤務状況コード（TBM_CODE.WORK_STATUS_KBN）

| コード   | ラベル   | カウント対象 |
| -------- | -------- | ------------ |
| 出勤     | 出勤     | workDays     |
| 有給     | 有給     | paidLeave    |
| 欠勤     | 欠勤     | absence      |
| 公休     | 公休     | publicHoliday |
| 特別休暇 | 特別休暇 | specialLeave |
| その他   | その他   | other        |

### 3.3 TbmRouteGroupCalendar（便カレンダー/委託パターン）

| カラム名        | 型       | 必須 | 説明               |
| --------------- | -------- | ---- | ------------------ |
| id              | Int      | ○    | 主キー（自動採番） |
| tbmRouteGroupId | Int      | ○    | 便ID（FK）         |
| date            | DateTime | ○    | 対象日             |
| active          | Boolean  | ○    | 稼働日フラグ       |
| createdAt       | DateTime | ○    | 作成日時           |
| updatedAt       | DateTime | ○    | 更新日時           |

#### ユニーク制約

- `unique_tbmRouteGroupId_date`: 同一便・同一日に1レコードのみ

## 4. チェックリスト

### 導入時の確認事項

- [ ] 営業所設定で便マスタが登録されているか
- [ ] 営業所設定でドライバーマスタが登録されているか
- [ ] 営業所設定で車両マスタが登録されているか
- [ ] 便の委託パターンが設定されているか（任意）

### 運用時の注意事項

- [ ] 重複配車（同一ドライバーが同日複数便）がないか定期確認
- [ ] 車両・ドライバー未設定の配車がないか確認
- [ ] 承認ステータスの進捗を確認
