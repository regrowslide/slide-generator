# 顧客の声分析システム（Hakobun Analysis）

生成AIを活用した顧客フィードバックの構造化・カテゴリ分類システムです。不定形の顧客の声を、組織固有の分析軸に合わせて構造化し、データ化します。

## 概要

本システムは、顧客からのフィードバックテキストを以下のプロセスで分析・構造化します：

1. **入力**: 生のテキストデータ（レビュー、アンケート、コメントなど）
2. **分析**: AIによるトピック単位分割、カテゴリ分類、感情分析、熱量スコア算出
3. **出力**: 構造化されたJSONデータ
4. **改善**: 人間による修正フィードバックを蓄積し、AIの精度を継続的に向上

### システムコンセプト

> **「新人の料理人（AI）に、秘伝のレシピ帳（DB）を持たせ、厨房（API）で働かせる」**

組織固有の判断基準（カテゴリマスター、ルール、修正事例）をデータベースに蓄積し、動的にプロンプトに組み込むことで、汎用的なAIを組織専属のアナリストへと進化させます。

## クイックスタート

### 1. クライアントの作成

まず、分析対象となる組織（クライアント）を作成します。

```
/hakobun/master/clients
```

- **クライアントID**: 半角英数字とアンダースコア（例: `cafe_sample_01`）
- **クライアント名**: 表示名（例: `サンプルカフェ`）

### 2. カテゴリマスターの登録

分析に使用するカテゴリを登録します。

```
/hakobun/master/categories
```

- **カテゴリコード**: 一意の識別子（例: `cat_01`）
- **大カテゴリ**: 上位分類（例: `店内`）
- **小カテゴリ**: 具体的なカテゴリ名（10〜16文字推奨、例: `オシャレ・雰囲気が良い`）
- **説明**: カテゴリの詳細説明（任意）

### 3. ルールの設定（任意）

AIの判断基準となるルールを設定します。

```
/hakobun/master/rules
```

- **対象カテゴリ**: ルールが適用されるカテゴリ
- **ルール説明**: 具体的な判断基準
- **優先度**: High / Medium / Low

### 4. テキスト分析の実行

メインダッシュボードでテキストを分析します。

```
/hakobun
```

1. クライアントを選択
2. 分析対象のテキストを入力
3. 「分析実行」ボタンをクリック
4. 結果を確認

### 5. フィードバックの送信

分析結果に誤りがある場合、修正してフィードバックを送信します。

1. 「フィードバック編集」タブに移動
2. 誤っている項目を修正
3. 「フィードバック送信」ボタンをクリック

送信されたフィードバックは次回以降の分析に反映され、AIの精度が向上します。

## 主な機能

### 📊 テキスト分析

- **トピック分割**: テキストを意味のまとまり（トピック単位）で分割
- **カテゴリ分類**: マスタカテゴリに基づく自動分類
- **感情分析**: 好意的 / 不満 / リクエストの判定
- **熱量スコア**: 0-100のスコアで感情の強さを数値化
- **ポジネガ判定**: positive / negative / neutral

### 🔄 フィードバックループ

- **修正データの蓄積**: 人間による修正を即座にデータベースに保存
- **Few-Shot学習**: 直近の修正事例をプロンプトに組み込み
- **ルール圧縮**: 修正事例から普遍的なルールを抽出（将来実装予定）

### 📁 マスタ管理

- **クライアント管理**: 複数組織のデータを分離管理
- **カテゴリ管理**: 組織固有のカテゴリ体系を構築
- **ルール管理**: AIの判断基準を明示的に定義

## ワークフロー

### 基本的なワークフロー

```
1. セットアップ
   ├─ クライアント作成
   ├─ カテゴリマスター登録
   └─ ルール設定（任意）

2. 分析実行
   ├─ テキスト入力
   ├─ AI分析
   └─ 結果確認

3. フィードバック
   ├─ 結果の確認・修正
   ├─ フィードバック送信
   └─ 次回分析への反映
```

## API エンドポイント

### 分析API

```typescript
POST /api/hakobun/analyze
{
  "client_id": "cafe_sample_01",
  "raw_text": "分析対象のテキスト",
  "voice_id": "optional_unique_id",
  "timestamp": "2025-12-16T10:00:00Z"
}
```

### フィードバックAPI

```typescript
POST /api/hakobun/feedback
{
  "client_id": "cafe_sample_01",
  "voice_id": "voice_id_from_analysis",
  "corrections": [
    {
      "extract_index": 0,
      "original_sentence": "元のトピック",
      "correct_category": "修正後のカテゴリ名",
      "correct_sentiment": "好意的",
      "reviewer_comment": "修正理由（任意）"
    }
  ]
}
```

### マスタ管理API

- `GET/POST/PUT/DELETE /api/hakobun/clients` - クライアント管理
- `GET/POST/PUT/DELETE /api/hakobun/categories` - カテゴリ管理
- `GET/POST/PUT/DELETE /api/hakobun/rules` - ルール管理
- `GET /api/hakobun/feedback` - 修正事例取得

## データベーススキーマ

主要なテーブル：

- `HakobunClient` - クライアント（組織）
- `HakobunCategory` - カテゴリマスター
- `HakobunRule` - ルール一覧
- `HakobunCorrection` - 修正データペア
- `HakobunVoice` - 分析結果保存

詳細は `prisma/schema/hakobun.prisma` を参照してください。
