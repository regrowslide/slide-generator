# Earth アプリ — ページ・機能一覧（不動産管理モジュール）

> 2024/03/04 MTG をもとに整理。設計計画は含まない。

---

## ユーザーロール

| ロール | 説明 |
|--------|------|
| 管理者（社内） | 全物件・全データにアクセス可能 |
| 担当者（社内） | 自分が担当する物件のデータにアクセス |
| オーナー | 自分の所有物件のみ閲覧。チャット・ファイル閲覧が可能 |
| 修繕業者 | 依頼された修繕案件のみ閲覧。対応記録の入力が可能 |

---

## 1. 社内向け画面

### 1-1. ダッシュボード

- 自分が担当する物件の **要対応事項**（未完了タスク・期限切れ）
- 進捗管理：進行中の案件一覧（誰が・何を・いつまで）
- 売上・営業成績の推移（月間 / 年間）
- オーナーからの未読チャット通知

### 1-2. オーナー管理

- オーナー一覧（検索・フィルタ）
- オーナー詳細
  - 基本情報（氏名 / 法人名、連絡先、契約種別：受託管理 or サブリース）
  - 所有物件一覧
- オーナー新規登録 / 編集

### 1-3. 物件管理

- 物件一覧（検索・フィルタ：受託管理 / サブリース / オーナー別）
- 物件詳細
  - 基本情報（名称、住所、種別、戸数、空室数、オーナー）
  - 担当者設定（複数人可）
- 物件新規登録 / 編集

### 1-4. 物件別ワークスペース

物件詳細から遷移。物件に紐づく以下の機能をタブ等で集約する。

#### 1-4a. 要対応事項（タスク管理）

> MTGの結果：タスク + トラブル対応を **1つのリスト「要対応事項」** に統合する。

- 一覧表示（ステータス：未着手 / 進行中 / 完了）
- 新規追加：タイトル、内容、担当者、期限、優先度
- 完了チェック
- 未完了・期限切れがダッシュボードに表示される
- 物件横断での一覧表示も可能（自分の担当物件をまたいで確認）

#### 1-4b. 修繕記録

> MTGの結果：修繕は要対応事項とは **別の専用機能** として残す。物件の資産管理として履歴を蓄積する。
> **業者依頼を使わずに社内だけで記録を残すことも、業者への依頼機能と連携させることも可能** なフレキシブルな設計とする。

- 一覧表示（時系列）
- 新規登録：
  - カテゴリ（水漏れ / 補修 / 電気 / その他）
  - 内容（自由記述）
  - 添付ファイル（画像等）
  - 期日
  - ステータス（未着手 / 対応中 / 完了）
  - 対応記録（対応日、対応画像、コメント）
- **業者依頼（任意）**：修繕記録から業者への依頼を作成することもできる
  - 依頼先の修繕業者を選択
  - 依頼送信（→ 業者側画面に通知）
  - 業者からの見積もり・対応記録が修繕記録に紐づく
  - 業者依頼なしでも修繕記録として完結可能

#### 1-4c. 社内チャット

> MTGの結果：チャットワークを廃止し、アプリ内の **物件ごとのチャットルーム** に置き換える。

- 物件に割り当てられた担当者のみ参加（管理者は全ルーム閲覧可）
- テキストメッセージ送受信
- ファイル共有（画像、PDF 等）
- チャット内容は物件ワークスペースからアクセス

#### 1-4d. ファイル保管

> MTGの結果：書類管理ではなく、物件ごとに **ファイルを保存・閲覧** できるシンプルな機能。
> ストレージは **Vercel Blob** を使用。

- ファイルアップロード
  - 対応形式：PDF / 画像（JPG, PNG等） / 動画（MP4等） / Word（.docx） / Excel（.xlsx） / その他一般的なファイル形式
  - 用途例：契約書、申込書、写真、動画、その他のメモ
- ファイル一覧表示・ダウンロード
- ブラウザ上で直接開ける（PDF・画像等）

#### 1-4e. オーナーチャット

> MTGの結果：公式LINEを置き換え。オーナーとの **チャット + ファイル共有**。

- テキストメッセージ送受信
- ファイル送受信（添付ファイルがアプリ上に残る）
- 送信予約機能（LINEにない機能として要望あり）

#### 1-4f. 入居者チャット

- 入居者との連絡用チャット
- テキストメッセージ送受信
- ファイル送受信

### 1-5. 修繕業者管理

- 修繕業者一覧
- 業者登録 / 編集（業者名、連絡先等）
- 業者へのアカウント発行

### 1-6. 入居者管理

- 入居者一覧（物件・部屋ごと）
- 入居者詳細（氏名、契約期間、ステータス：入居中 / 退去予定 / 空室）
- 入居者へのアカウント発行

## 2. オーナー向けポータル

> 専用ログインページからアクセス。自分の所有物件のみ閲覧可能。

### 2-1. オーナーダッシュボード

- 所有物件一覧
- 未読メッセージ通知

### 2-2. 物件詳細（オーナー視点）

- 物件基本情報の閲覧
- 共有ファイルの閲覧・ダウンロード
- 管理会社とのチャット（= 社内の 1-4e と対になる画面）

---

## 3. 修繕業者向けポータル

> 専用ログインページからアクセス。依頼された修繕案件のみ閲覧可能。

### 3-1. 修繕依頼一覧

- 自社宛の修繕依頼を一覧表示
- ステータスフィルタ（依頼中 / 受注 / 完了）

### 3-2. 修繕依頼詳細

- 物件情報
- カテゴリ / 内容 / 添付ファイル / 期日
- 見積もり送信（画像 or ファイル + メッセージ）
- 対応記録の入力
  - 予定日
  - ステータス更新（受注 / 完了）
  - 対応記録画像アップロード
  - コメント
- 管理会社とのメッセージ機能

## 4. 横断機能

| 機能 | 説明 |
|------|------|
| 通知 | 要対応事項の期限切れ、チャットの未読、修繕依頼の更新等をダッシュボード + メール等で通知 |
| 物件横断タスク表示 | 複数物件をまたいで自分の要対応事項を一覧で確認 |
| レスポンシブ対応 | スマホ・タブレットでも利用可能（MTGで確認済み） |
| Google カレンダー連携 | タスク・予定をGoogleカレンダーに反映（将来検討） |
| Gmail 通知連携 | 重要な通知をGmailに送信（将来検討） |

## 5. 現モックからの変更点まとめ

| 現モック | MTG後の方針 |
|----------|-------------|
| 書類管理（提出済/未提出/期限切れ） | → 物件ごとのファイル保管（シンプルなアップロード・閲覧） |
| タスク管理 + トラブル事例を別々に管理 | → **要対応事項** として統合。1つのリストで管理 |
| トラブル事例共有 | → 要対応事項に統合（カテゴリで区別） |
| チャット（オーナー向けのみ） | → オーナーチャット + **社内チャット（物件ごと）** + 入居者チャット を追加 |
| 修繕業者との連携なし | → **修繕履歴** 専用機能 + **修繕業者ポータル** を新設 |
| オーナーポータルなし | → **オーナー向けポータル** を新設 |
| ダッシュボード未定義 | → 自分担当の要対応事項 + 売上推移 + 未読通知 |
