## 1. システム概要

### 1-1. Earthとは

Earth は、賃貸不動産の管理業務を一元化するシステムです。物件ごとの「要対応事項」「修繕記録」「チャット」「ファイル保管」を集約し、社内スタッフ・オーナー・修繕業者がそれぞれ必要な情報にアクセスできます。

### 1-2. 利用者と権限

| 利用者 | できること |
|--------|-----------|
| 管理者（社内） | すべての画面・データにアクセス可能 |
| 担当者（社内） | 自分が担当する物件のデータを操作 |
| オーナー | 自分の所有物件の閲覧、管理会社とのチャット、共有ファイルの閲覧 |
| 修繕業者 | 依頼された修繕案件の閲覧、見積もり・対応記録の入力 |

### 1-3. 画面構成

```
┌──────────────────────────────────────────────────────┐
│  ヘッダー（Earth ロゴ + ポータル切替）                │
├──────────┬───────────────────────────────────────────┤
│          │                                           │
│  サイド   │       メインコンテンツ                    │
│  バー     │                                           │
│          │  ┌─────────────────────────────────────┐  │
│ ─────── │  │  ダッシュボード / 各管理画面         │  │
│ ダッシュ  │  │  / ワークスペース                    │  │
│ ボード   │  │                                     │  │
│ ─────── │  └─────────────────────────────────────┘  │
│ オーナー  │                                           │
│ 管理     │                                           │
│ ─────── │                                           │
│ 物件管理  │                                           │
│ ─────── │                                           │
│ 修繕業者  │                                           │
│ 管理     │                                           │
│ ─────── │                                           │
│ 入居者   │                                           │
│ 管理     │                                           │
│          │                                           │
├──────────┴───────────────────────────────────────────┤
```

**社内画面の構成:**

| # | ページ | 説明 |
|---|--------|------|
| 1 | ダッシュボード | 要対応事項サマリー、売上推移、未読通知 |
| 2 | オーナー管理 | オーナーの一覧・登録・編集・削除 |
| 3 | 物件管理 | 物件の一覧・登録・編集・削除 |
| 4 | 物件別ワークスペース | 物件に紐づく6つの機能をタブで集約 |
| 5 | 修繕業者管理 | 修繕業者の一覧・登録・編集・削除 |
| 6 | 入居者管理 | 入居者の一覧表示 |

**外部ポータル:**

| # | ポータル | 説明 |
|---|---------|------|
| 7 | オーナーポータル | オーナー自身の所有物件情報・チャット・ファイル閲覧 |
| 8 | 修繕業者ポータル | 修繕依頼の一覧・詳細・対応記録入力 |

### 1-4. データの関係

```
オーナー ──1:N──▶ 物件 ──1:N──▶ 入居者
                    │
                    ├──1:N──▶ 要対応事項
                    ├──1:N──▶ 修繕記録 ──0:1──▶ 修繕依頼 ──▶ 修繕業者
                    ├──1:N──▶ チャット（社内 / オーナー / 入居者）
                    └──1:N──▶ ファイル
```

- 1人のオーナーが複数の物件を所有できます
- 物件ごとに要対応事項・修繕記録・チャット・ファイルが紐づきます
- 修繕記録から業者への依頼を作成できます（作成しなくてもOK）

## 2. 初回セットアップ

システムを使い始めるには、以下の順番でデータを登録してください。

### 手順

| 順番 | 操作 | 画面 |
|------|------|------|
| 1 | 修繕業者を登録する | 修繕業者管理 |
| 2 | オーナーを登録する | オーナー管理 |
| 3 | 物件を登録する（オーナーを紐づけ） | 物件管理 |
| 4 | 物件のワークスペースで必要なファイルをアップロード | 物件別ワークスペース → ファイル保管 |

> **ポイント**: 物件を登録する前にオーナーを登録しておく必要があります。物件登録時にオーナーを選択するためです。

## 3. ダッシュボード

ログイン後に最初に表示される画面です。担当物件の状況を一目で把握できます。

### 3-1. サマリーカード（4つ）

| カード | 表示内容 |
|--------|---------|
| 未完了タスク | ステータスが「完了」以外の要対応事項の件数 |
| 期限切れ | 期限を過ぎた未完了タスクの件数 |
| 未読チャット | オーナーからの未読メッセージ件数 |
| 管理物件 | 管理している物件の総数 |

### 3-2. 売上推移チャート

直近8ヶ月分の月間売上を棒グラフで表示します。各棒の上に金額（万円単位）が表示されます。

### 3-3. 要対応事項リスト

未完了の要対応事項を期限が近い順に最大8件表示します。

- **赤枠表示**: 期限を過ぎた項目
- **クリック**: その物件のワークスペースに遷移

各項目には以下の情報が表示されます:
- 優先度（高・中・低）
- タイトル
- 物件名・担当者名
- 期限日
- ステータス

## 4. オーナー管理

物件オーナーの情報を管理する画面です。

### 4-1. オーナー一覧

| 列 | 説明 |
|----|------|
| オーナー名 | クリックすると詳細を表示 |
| 区分 | 個人 / 法人 |
| 電話番号 | （PC表示時のみ） |
| 契約タイプ | 受託管理 / サブリース |
| 物件数 | 所有物件の件数 |
| 操作 | 編集・削除ボタン |

**絞り込み機能:**
- テキスト検索（オーナー名・電話番号・メールアドレス）
- 契約タイプで絞り込み

### 4-2. オーナー詳細（クリック時）

オーナー名をクリックすると、以下の情報がポップアップ表示されます。

- **基本情報**: 区分、契約タイプ、電話番号、メールアドレス
- **所有物件一覧**: 物件名・住所・総戸数・空室数
  - 各物件に **「ワークスペース」** ボタンがあり、クリックするとその物件のワークスペースに直接遷移できます

### 4-3. オーナーの新規登録・編集

「新規追加」ボタンまたは一覧の編集ボタンをクリックすると入力画面が表示されます。

| 項目 | 必須 | 説明 |
|------|------|------|
| オーナー名 | ○ | 氏名または法人名 |
| 区分 | ○ | 個人 / 法人 |
| 契約タイプ | ○ | 受託管理 / サブリース |
| 電話番号 | - | ハイフン付きで入力 |
| メールアドレス | - | 連絡用メールアドレス |

### 4-4. オーナーの削除

一覧の削除ボタンをクリックすると、確認画面が表示されます。

> **注意**: オーナーを削除すると、物件との紐づけが解除されます。物件データ自体は残りますが、オーナー情報が参照できなくなります。

|------|------|------|

## 6. 物件別ワークスペース

物件管理で物件名をクリックすると開く画面です。その物件に関するすべての業務情報が**6つのタブ**に集約されています。

### 画面上部

- **「物件一覧に戻る」リンク**: 物件管理画面に戻ります
- **物件情報ヘッダー**: 物件名、住所、オーナー名、種別、戸数、担当者

### 6つのタブ

| タブ | 機能 |
|------|------|
| 要対応事項 | タスク管理（登録・完了チェック・フィルタ） |
| 修繕記録 | 修繕履歴の管理（業者依頼との連携あり） |
| 社内チャット | 担当者間のメッセージ |
| ファイル保管 | 契約書・写真等のファイル管理 |
| オーナーチャット | オーナーとのメッセージ（送信予約機能あり） |
| 入居者チャット | 入居者とのメッセージ |

> **スマートフォンでの操作**: タブは横スクロールで切り替えられます。タブのラベルはアイコンのみの表示になります。

---

### 6-1. 要対応事項タブ

物件に関するタスクやトラブル対応をひとつのリストで管理します。

#### 一覧表示

各項目に表示される情報:
- 完了チェックボタン（丸いボタン。クリックで完了/未完了を切り替え）
- タイトル
- カテゴリ・期限・担当者
- 優先度バッジ（高=赤 / 中=黄 / 低=青）
- 編集ボタン

**期限切れの項目**は赤い枠線で強調されます。

**ステータスフィルタ**: 「すべて」「未着手」「進行中」「完了」で絞り込み

#### 新規追加・編集

| 項目 | 必須 | 説明 | 入力例 |
|------|------|------|--------|
| タイトル | ○ | 対応内容の概要 | 外壁塗装の見積依頼 |
| カテゴリ | ○ | 修繕 / 募集 / 契約 / 報告 / 管理 / クレーム / その他 | 修繕 |
| 優先度 | ○ | 高 / 中 / 低 | 高 |
| 期限 | - | 日付選択 | 2026-03-15 |
| 担当者 | - | 担当者名を入力 | 佐藤 |
| 説明 | - | 詳細や補足 | 3社から見積もり取得中 |

---

### 6-2. 修繕記録タブ

物件の修繕履歴を蓄積する機能です。資産管理の観点で、いつ・どんな修繕が行われたかを記録します。

#### 一覧表示

各記録に表示される情報:
- カテゴリ（水漏れ / 補修 / 電気 / その他）
- ステータス（未着手 / 対応中 / 完了）
- 登録日
- 内容の概要
- 業者名（業者依頼がある場合）

#### 詳細表示（クリック時）

- カテゴリ、ステータス、期日、登録日
- 修繕内容
- 対応記録（時系列）: 日付とコメント

> **業者依頼について**: 修繕記録は社内だけで完結させることもできます。必要に応じて修繕業者への依頼を作成し、見積もりの受け取りや対応記録を業者と共有することもできます。

---

### 6-3. 社内チャットタブ

物件の担当者同士でメッセージを交換する機能です。チャットワーク等の外部ツールの代わりに、物件単位でやり取りを集約します。

- 自分のメッセージは右側（青い吹き出し）
- 他のスタッフのメッセージは左側（グレーの吹き出し）
- 各メッセージに送信者名と送信時刻が表示されます
- 画面下部のテキスト入力欄からメッセージを送信

---

### 6-4. ファイル保管タブ

物件に関する各種ファイルを保管・閲覧する機能です。

#### ファイル一覧

| 表示項目 | 説明 |
|---------|------|
| ファイル名 | ファイルの種類に応じたアイコン付き |
| サイズ | MB単位 |
| アップロード日 | 登録日 |
| アップロード者 | 登録したスタッフ名 |
| ダウンロード | ダウンロードボタン |

**対応ファイル形式:**
- PDF（契約書等）
- 画像（JPG, PNG — 物件写真等）
- 動画（MP4 — 内見動画等）
- Word（.docx）
- Excel（.xlsx）
- その他

#### アップロード

画面右上の「アップロード」ボタンからファイルを追加します。

---

### 6-5. オーナーチャットタブ

物件オーナーとのメッセージ機能です。公式LINEの代わりとして使用します。

- 基本的な操作は社内チャットと同じです
- **送信予約機能**: カレンダーアイコンをクリックすると、メッセージの送信日時を予約できます

> **オーナー側**: オーナーは「オーナーポータル」から同じチャットにアクセスできます（後述）。

---

### 6-6. 入居者チャットタブ

入居者との連絡用メッセージ機能です。

- 操作方法は社内チャットと同じです
- 入居者からの問い合わせ（設備故障の報告等）に対して返信できます

## 9. オーナーポータル

物件オーナー専用の画面です。自分の所有物件に関する情報だけが表示されます。

### 9-1. 所有物件一覧

ログイン後に表示される画面です。自分が所有する物件がカード形式で一覧表示されます。

各カードの表示内容:
- 物件名
- 住所
- 総戸数・空室数

**クリック**すると物件の詳細画面に遷移します。

### 9-2. 物件詳細

3つのタブで切り替えます。

| タブ | 内容 |
|------|------|
| 物件情報 | 住所、種別、総戸数、空室数 |
| 共有ファイル | 管理会社が共有したファイルの閲覧・ダウンロード |
| チャット | 管理会社とのメッセージ（社内の「オーナーチャット」と対になる画面） |

> **オーナーが見られるのは**: 自分の所有物件に関する情報のみです。他のオーナーの物件や社内チャットの内容は閲覧できません。

---

## 10. 修繕業者ポータル

修繕業者専用の画面です。自社宛に依頼された修繕案件のみが表示されます。

### 10-1. 修繕依頼一覧

自社宛の修繕依頼がカード形式で一覧表示されます。

各カードの表示内容:
- 修繕内容
- ステータス（依頼中 / 受注 / 完了）
- 物件名・カテゴリ・期日

### 10-2. 修繕依頼詳細

依頼をクリックすると詳細画面が表示されます。

表示内容:
- **物件情報**: 物件名
- **修繕情報**: カテゴリ、ステータス、期日、修繕内容
- **見積もり**: 提出済みの見積もり内容
- **対応記録**: 時系列の対応コメント

> **業者ができること**: 見積もりの送信、対応記録の入力（対応日、コメント、ステータス更新）

> **注意**: これはデモ用の切替機能です。本番環境ではログインアカウントに応じて自動的に適切な画面が表示されます。

## 12. 用語集

| 用語 | 説明 |
|------|------|
| 受託管理 | オーナーから物件の管理を委託される契約形態。オーナーが物件を所有したまま管理業務を依頼する |
| サブリース | 管理会社がオーナーから物件を一括で借り上げ、入居者に転貸する契約形態 |
| オーナー | 物件の所有者。個人と法人がある |
| 担当者 | 物件の管理を担当する社内スタッフ。1つの物件に複数人を割り当て可能 |
| 物件別ワークスペース | 物件に紐づく情報（タスク・修繕・チャット・ファイル等）を集約した作業画面 |
| 要対応事項 | タスクとトラブル対応を統合した管理項目。カテゴリ（修繕・募集・契約・クレーム等）で区別する |
| 修繕記録 | 物件の修繕履歴。資産管理の観点で蓄積する。業者への依頼とは独立して記録可能 |
| 修繕依頼 | 修繕記録から作成する業者への作業依頼。修繕記録1件に対して0件または1件の依頼を紐づけられる |
| 修繕業者 | 修繕工事を請け負う外部業者。専用ポータルから依頼内容の確認・対応記録の入力ができる |
| ポータル | 外部利用者（オーナー・修繕業者）向けの専用画面。社内データのうち、その利用者に関係する情報のみが表示される |
| 送信予約 | オーナーチャットで利用可能な機能。メッセージの送信日時を事前に指定して予約送信する |
| ファイル保管 | 物件ごとにファイルを保存・閲覧する機能。対応形式: PDF、画像、動画、Word、Excel 等 |
| 対応記録 | 修繕記録や修繕依頼において、対応した日付とコメントを時系列で記録するもの |
