# 「できたよ！」アプリ — 仕様書

## 1. プロダクト概要

### 1.1 コンセプト
子どもの「できた！」を可視化し、日常の生活習慣と成長をゲーム感覚で楽しく身につける育児支援アプリ。
親の「何度言ってもやらない」「どこまでできるようになったか分からない」という悩みを解決する。

### 1.2 ターゲットユーザー
- 主操作者: 2〜6歳の子ども（親と一緒に操作）
- 管理者: 保護者（30代前後の父母）

### 1.3 基本思想
- 「叱る→やらせる」ではなく「できた→嬉しい→もっとやりたい」のポジティブループ
- 毎日繰り返す「ルーチン」と、一度できたら身につく「スキル」を明確に分離
- 達成履歴は永久保存し、子どもの成長の記録として蓄積する

### 3.5 トグル動作

- タップ1回目: 未完了 → 完了
  - 完了エフェクト発火（S-04ポップアップ表示）
  - 達成履歴に1件追加（永久保存）
  - チェックマークアニメーション（checkPop）
- タップ2回目: 完了 → 未完了
  - エフェクトなし（静かに戻る）
  - 達成履歴は削除しない（一度得たバッジは永久保持）

### 3.6 カテゴリヘッダー


- 右: 完了数 / 合計（10px, グレー）
- 下線: `rgba(0,0,0,0.05)` 1px

### 3.7 プログレスバー

- 位置: カード一覧の上部
- 高さ: 20px, 角丸20px
- 背景: `#C8E6C9`
- バー色: `linear-gradient(90deg, #2ED573, #6BCB77, #A8E6CF)`
- テキスト: `{完了数} / {合計}` 中央配置
- アニメーション: shimmerBar（バーが輝く）

### 3.8 連続日数表示

- 位置: プログレスバー右横
- 表示: `🔥{N}日`
- アニメーション: streakGlow（テキストシャドウが点滅）
- ※ v1ではモック値。将来的にストレージ連動

### 3.9 コンプリートバナー

全ルーチン完了時に表示。

- テキスト: `🎉 きょうのルーチンコンプリート！`
- 背景: グリーングラデーション + 緑ボーダー
- アニメーション: fadeUp

### 3.10 リセットボタン

- 表示条件: 1つ以上完了時
- テキスト: `🔄 きょうのリセット`
- 動作: 全ルーチンの完了状態を解除（達成履歴は残す）
- 位置: カード一覧の最下部

## 5. S-03 じっせきかくにん画面

### 5.1 概要
これまでに獲得した全バッジの累計コレクションと、日別の達成タイムラインを表示する全画面オーバーレイ。

### 5.2 遷移
- 入口: ヘッダー右上の 🏆 ボタン（累計バッジ数を表示）
- 出口: 左上の ← ボタン

### 5.3 バッジコレクション

画面上部に白カード。獲得した全バッジをグリッド表示。

| 項目 | 仕様 |
|------|------|
| レイアウト | flexbox wrap |
| 1バッジサイズ | 44x44px, 角丸12px |
| バッジ背景 | `#FFFDE7→#FFF8E1` グラデーション |
| アニメーション | fadeUp（0.02s間隔でスタガー、最大0.5s） |
| 空状態 | 「まだバッジがありません。ルーチンやスキルをクリアしよう！」 |

### 5.4 日別タイムライン

バッジコレクションの下に日付ごとにグルーピング表示。

```
📅 きょう                          3コ
────────────────────────────────────────
[🌟 じかんにおきる 🔄] [⭐ おはようと言う 🔄] [✨ かおをあらう 🔄]

📅 03/10                           12コ
────────────────────────────────────────
[🌟 じかんにおきる 🔄] [⭐ おはようと言う 🔄] ...
```

各エントリーの構成:
- Sticker emoji（18px）
- 名前テキスト（11px, bold）
- 種別アイコン: ルーチン=🔄 / スキル=🗺️（9px）

### 5.5 達成履歴データ構造

```typescript
interface AchievementEntry {
  sticker: string;    // バッジのemoji
  name: string;       // アイテム名
  type: "routine" | "skill";  // 種別
  date: string;       // ISO日付 "YYYY-MM-DD"
  ts: number;         // Unix timestamp（ソート用）
}
```

### 5.6 永続化

- ストレージ: localStorage（キー: `dekitayo_history`）
- 形式: JSON配列
- 書き込みタイミング: アイテムをONにした瞬間
- 削除: トグルOFFしても履歴は削除しない
- 将来: サーバーサイドDB移行を想定

## 6. S-04 完了エフェクト

### 6.1 トリガー
カードを未完了 → 完了に切り替えた瞬間に発火。完了 → 未完了では発火しない。

- `radial-gradient` で中心から光が広がる

- アニメーション: burstOut（中心から放射状に飛散）
- duration: 0.5〜1.1s

#### 6.2.3 二次バーストリング
- 個数: 20個
- 色: `#FFD93D` (金色の光点)
- 光彩: `box-shadow: 0 0 6px 2px rgba(255,217,61,0.5)`

#### 6.2.4 スティッカーレイン

#### 6.2.5 エクスパンディングリング

## 9. ビジネスロジック

### 9.1 ルーチンリセット

- 手動リセット: 「きょうのリセット」ボタンで全ルーチンの完了状態を解除
- 自動リセット（v2）: 日付が変わったら自動でリセット
- リセット時に達成履歴は削除しない

### 9.2 連続日数カウント

```
ルーチン全完了した日の翌日にも全完了 → streak++
1日でも全完了できなかった → streak = 0
```

### 9.3 スキル「スーパーキッズ」自動解放（v2）

- 他の全スキル（s01〜s17）が完了状態 → s18を自動的に完了に
- いずれかがOFFに戻された → s18も自動的にOFFに

### 9.4 達成履歴の記録ルール

- 同一アイテムを同日に何度ON/OFFしても、ONにするたびに1件記録
- 履歴はappend-only（削除機能なし、v1時点）
- 将来的に「1日1回のみ記録」に制限するオプションを検討

| 高 | カスタムアイテム追加 | 親がルーチン/スキルを自由に追加・編集 |
| 中 | 家族共有 | 複数の子ども切替、パートナーとの共有 |
| 中 | ごほうびシステム | N日連続達成→特別バッジ / ごほうび設定 |
| 中 | 週間・月間レポート | 達成率グラフ、よくできた項目ランキング |
| 低 | アバター育成 | 達成に応じてキャラクターが成長 |
| 低 | 音声対応 | 項目名の読み上げ（ひらがなが読めない年齢向け） |
| 低 | テーマ切替 | キャラクターテーマ、季節テーマ |

## 12. 設計上の注意事項

### 12.1 子ども向けUI原則
- テキストは全てひらがな（漢字は使わない）
- タップ領域は最低40x40px
- 色のコントラストは十分に確保
- 音が出る場合はミュート可能に

### 12.2 親の負担軽減
- セットアップ不要ですぐ使える（デフォルトアイテム付き）
- 1タップで完了/未完了切替
- 「叱る」ではなく「褒める」設計に徹する

### 12.3 データの扱い
- 子どもの個人情報は最小限（名前すら必須にしない）
- 達成データの外部送信は行わない（v1）
- COPPA/個人情報保護への配慮
