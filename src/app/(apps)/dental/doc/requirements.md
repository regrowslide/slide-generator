# 訪問歯科アプリ 要件定義書

> **最終更新**: 2026-02-06
> **実装状況**: モック完成（`doc/mock/DentalAppMock.jsx`）

## 1. プロジェクト概要

訪問歯科診療を行う歯科医師および歯科衛生士向けの、スケジュール管理・診療記録・点数管理サポートアプリケーション。

### 目的

- 複雑な診療報酬点数の計算サポート
- 訪問先（施設）での効率的な回診順序の管理
- 提供文書作成の省力化

### ターゲットユーザー

- 歯科医師
- 歯科衛生士

### 利用シーン

- 訪問先（特別養護老人ホームなどの施設）でのタブレット利用を想定

---

## 2. ユーザー課題（ヒアリングに基づく）

| 課題         | 詳細                                                             |
| ------------ | ---------------------------------------------------------------- |
| 回診順序管理 | 施設内の回診順序（フロアや部屋）の管理と並び替えが煩雑           |
| 診療時間管理 | 診療時間（20分ルールなど）の管理と記録が必要                     |
| 記録の効率化 | SOAPカルテ（経過記録、処置内容）や口腔内状態の記録を効率化したい |
| 申し送り確認 | 前回診療内容（申し送り）を即座に確認したい                       |
| 並行作業対応 | 医師と衛生士が並行して動く場合や、単独で動く場合に対応したい     |
| 算定漏れ防止 | 前回の算定日を参考に算定提案をしてほしい                         |

### 3.3 各画面の詳細

---

#### A. ダッシュボード（トップ画面） ✅

**ページID:** `dashboard`

**概要:** アプリの入口。主要機能へのナビゲーションと本日のサマリーを表示

**UI要素:**

- 統計サマリー（本日の訪問予定数、診察完了数、今日の日付）
- 4つのナビゲーションカード:
  - Schedule / Visits（訪問計画スケジュール）
  - Patient Management（利用者マスタ）
  - Individual Input（個別入力）
  - Master Data Management（マスタデータ管理）

---

#### B. 訪問計画スケジュール（カレンダー） ✅

**ページID:** `schedule`

**概要:** 月間カレンダー形式で訪問計画を管理

**UI要素:**

- 月間カレンダー表示（月切り替え可能）
- 施設フィルター（ドロップダウン）
- 「+ 新規作成」ボタン
- 予定がある日に施設名ラベル表示
- 予定クリック → 訪問計画詳細へ遷移

---

#### C. 訪問計画詳細（並び替え・診察前設定） ✅

**ページID:** `visit-detail`

**概要:** 当日の訪問対象者の選択・診察順序の設定

**UI要素（2カラムレイアウト）:**

| 左カラム                       | 右カラム                   |
| ------------------------------ | -------------------------- |
| **施設登録患者リスト**         | **本日の訪問・診察リスト** |
| - 建物×フロアでグループ化      | - 診察対象者カード         |
| - クリック/+ボタンで追加       | - 担当医/DH選択            |
| - チェックマークで選択状態表示 | - 診察(医)/指導(衛)ボタン  |

**ヘッダー:** 日付、施設名、「訪問全体を終了する」ボタン

---

#### D. 診療画面 ✅

**ページID:** `consultation`

**概要:** 診察中の記録入力画面（医師モード/衛生士モード）

**DHモード制限:** DHログイン時は「訪衛指」と「在口衛」のみ操作可能（MTG 0206決定）

**セクション:**

1. ヘッダー（患者情報、担当者、モード表示 ※DHモード時はバナー表示）
2. タイマー（DR/DH別、20分アラート付き）
3. バイタル測定（処置前/処置後）
4. ~~本日の実施項目~~ ⬅︎ 一旦廃止（MTG 0206決定）
5. 実施記録・所見（4項目: 訪問時の様子/口腔内所見/処置/次回予定）
6. 実施項目の選択（加算）（ON/OFFトグル、該当区分選択、自動判定、点数計算）
   - 各項目の「i」アイコンをクリックでテキストポップアップ表示（PDFリンクではない）

---

#### E. 最終提示画面 ✅

**ページID:** `final-review`

**概要:** 診療完了後のサマリー確認

**UI要素:**

- 診療情報（患者、施設、日付、担当者、診療時間）
- 実施記録
- 算定項目と合計点数

---

#### F. 利用者マスタ（統合版） ✅

**ページID:** `admin-patients`

**概要:** 全患者の検索・閲覧・追加・編集・削除を統合した管理画面

**UI要素:**

- ヘッダー: 「利用者マスタ」タイトル + 「利用者を追加」ボタン
- フィルター行: 検索バー（名前/カナ） + 施設フィルタドロップダウン
- 患者テーブル（#、氏名、ふりがな、施設、居場所、年齢、申し送り、操作）
- 操作ボタン: 詳細→patient-detail、編集→patient-edit、削除（確認あり）
- 追加/簡易編集はPatientFormモーダル（useModal使用）

---

#### G. 患者認識・アセスメント（詳細） ✅

**ページID:** `patient-detail`

**概要:** 患者の全プロフィール情報を閲覧

**UI要素:**

- 基本情報（氏名、施設、居室、年齢、性別、介護度）
- 訪問先情報
- 既往歴・疾患
- 口腔状態
- 算定履歴
- 提供文書
- 経過記録
- 「編集」ボタン → 患者編集ページへ遷移

---

#### H. 患者情報入力・編集 ✅

**ページID:** `patient-edit`

**概要:** 患者の詳細情報とアセスメント入力

**左パネル（患者情報入力）:**

- 患者ID、姓/名、生年月日、年齢（自動算出）
- 基礎疾患チェックボックス（14項目: 認知症、高血圧症、脳血管障害、精神疾患、パーキンソン病、心不全、末期がん、老衰、大腿骨頸部骨折、脊柱管狭窄症、ALS、腎臓小脳変性症、多発性硬化症、廃用症候群）
- セイ/メイ（必須）
- お薬情報（お薬手帳画像の保存）
- 診療情報等連携共有料（情共1）チェック + 最終取得日
- 総合医療管理加算（総医）チェック + 最終取得日

**右パネル（患者状態・アセスメント）:**

- 基本情報: 身長、体重、BMI（自動計算）
- 誤嚥性肺炎の既往（無し/あり + 日付、繰り返しあり + 日付）
- 身体・口腔状況: 座位保持(JZ3-1)、口腔清掃の状況、お口の中の水分保持、うがい
- リスク・摂取: 低栄養リスク、むせ、経口摂取の有無
- 栄養・水分: 人工栄養法、水分
- 食事・服薬: 食形態（主食JZ12-1～3/おかずJZ12-4～5）、飲み込み、薬の服用

---

#### I. 個別入力 ✅

3. 届出・施設基準（チェックボックス形式）

#### O. 施設マスタ ✅

**ページID:** `admin-staff`

**概要:** 歯科医師/歯科衛生士の管理（CRUD + 並び替え）

#### R. 日次報告 ✅

**ページID:** `summary`

**概要:** 本日の診療完了後のサマリー表示

**テーブル列:** 患者名、施設、実施項目、合計点数、文書状況（済/未）、ステータス

---

#### S. 履歴・一括印刷 ✅（モック）

**ページID:** `batch-print`

**概要:** 施設×月でフィルタし、診療履歴をまとめてPDF一括出力

- 一括PDF出力ボタン

- 診療履歴テーブル（日付、患者名、施設、実施項目、点数、文書状況）

#### T. 施設ポータル画面（施設側） ✅

**ページID:** `facility-portal`

**概要:** 施設職員が利用者のアセスメント情報を入力・更新する画面。施設ポータル管理（ID/PW発行）は施設マスタに統合済み。

**UI要素:**

- 施設名ヘッダー + ログアウトボタン
- 利用者一覧（アセスメント入力ボタン付き）
- アセスメント編集フォーム（PatientEditPageと同等の項目）:
  - 身長/体重/BMI（自動計算）
  - 介護度、誤嚥性肺炎既往、座位保持、口腔清掃自立度
  - 保湿剤使用、含嗽、低栄養リスク、むせ
  - 経口摂取、人工的栄養、水分形態、主食/副食形態
  - 嚥下機能、内服嚥下
  - 基礎疾患チェックボックス（14項目）
  - 残歯数、義歯有無、口腔機能低下症
  - 備考

## 4. データモデル

### 4.1 主要定数

#### 届出・施設基準（CLINIC_QUALIFICATIONS）

| ID         | 名称                     | 説明                              |
| ---------- | ------------------------ | --------------------------------- |
| shiensin1  | 歯援診1                  | 在宅療養支援歯科診療所1           |
| shiensin2  | 歯援診2                  | 在宅療養支援歯科診療所2           |
| zahoshin   | 在補診                   | 在宅療養支援歯科診療所（補完）    |
| koukukan   | 口腔管                   | 口腔機能管理料の施設基準          |

| other      | その他                   | テキスト入力付き                  |

> 全項目boolean型（チェックボックス）。`other`のみ追加テキストフィールド（`otherText`）あり

#### 基礎疾患（PATIENT_DISEASES）

認知症、高血圧症、脳血管障害、精神疾患、パーキンソン病、心不全、末期がん、老衰、大腿骨頸部骨折、脊柱管狭窄症、筋萎縮性側索硬化症（ALS）、腎臓小脳変性症、多発性硬化症、廃用症候群（全14項目）

#### アセスメント項目（ASSESSMENT_OPTIONS）

| カテゴリ   | 項目               | 選択肢                                                                 |
| ---------- | ------------------ | ---------------------------------------------------------------------- |
| 身体・口腔 | 座位保持           | 良好 / やや不良 / 不良                                                 |
| 身体・口腔 | 口腔清掃の状況     | 自主的に行える / 声がけのみ必要 / 一部お手伝いが必要 / 全介助が必要    |
| 身体・口腔 | お口の中の水分保持 | 無理なく可能 / 困難 / 不可能(むせる) / 飲んでしまう / 口から出てしまう |
| 身体・口腔 | うがい             | 可能 / 困難 / 不可能（むせる） / 飲んでしまう / 口から出てしまう       |
| リスク     | 低栄養リスク       | なし / 少しあり / リスク高め                                           |
| リスク     | むせ               | なし / 液体で時々 / 頻繁にある                                         |
| リスク     | 経口摂取の有無     | 全て経口摂取 / 一部経口摂取 / 経口摂取なし                             |
| 栄養       | 人工栄養法         | 無し / 胃瘻 / 経鼻 / 抹消静脈 / 中心静脈 / その他                      |
| 栄養       | 水分               | トロミなし / トロミあり / 経口摂取禁止                                 |
| 食事       | 主食               | 常食 / 軟飯 / ペースト食                                               |
| 食事       | おかず             | 常食 / 一口大・軟菜 / 刻み食 / ミキサー食                              |
| 食事       | 飲み込み           | 問題なし / 時々むせることがある / 頻繁にむせてしまう                   |
| 服薬       | 薬の服用           | 問題なし / 苦手 / 上手く飲み込めない                                   |

## 6. UIデザイン方針

| 方針               | 詳細                                                                       |
| ------------------ | -------------------------------------------------------------------------- |
| タッチ操作最適化   | タブレットでの操作を想定し、ボタンサイズや入力エリアを大きめに確保         |
| 視認性             | 高齢者施設での利用も考慮し、コントラスト高めの配色                         |
| ステータス色分け   | 未＝グレー、中＝青、済＝緑、要算定/超過＝赤                                |
| 情報階層           | 常に「今誰を診ているか」「時間はどれくらい経過したか」がわかるヘッダー配置 |
| セクションヘッダー | 青背景に白文字で視認性を確保（届出・施設基準、患者情報入力等）             |
