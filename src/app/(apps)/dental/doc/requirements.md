# 訪問歯科アプリ 要件定義書

> **最終更新**: 2026-01-27
> **実装状況**: モック完成（`doc/mock/DentalAppMock.jsx`）

## 1. プロジェクト概要

訪問歯科診療を行う歯科医師および歯科衛生士向けの、スケジュール管理・診療記録・点数管理サポートアプリケーション。

### 目的

- 複雑な診療報酬点数の計算サポート
- 訪問先（施設）での効率的な回診順序の管理
- 提供文書作成の省力化

### ターゲットユーザー

- 歯科医師
- 歯科衛生士

### 利用シーン

- 訪問先（特別養護老人ホームなどの施設）でのタブレット利用を想定

---

## 2. ユーザー課題（ヒアリングに基づく）

| 課題 | 詳細 |
|------|------|
| 回診順序管理 | 施設内の回診順序（フロアや部屋）の管理と並び替えが煩雑 |
| 診療時間管理 | 診療時間（20分ルールなど）の管理と記録が必要 |
| 記録の効率化 | SOAPカルテ（経過記録、処置内容）や口腔内状態の記録を効率化したい |
| 申し送り確認 | 前回診療内容（申し送り）を即座に確認したい |
| 並行作業対応 | 医師と衛生士が並行して動く場合や、単独で動く場合に対応したい |

```

├── (pages)/
│   ├── schedule/           # 訪問計画スケジュール（カレンダー） ✅ モック実装済

│   │
│   ├── consultation/       # 診療画面 ✅ モック実装済
│   │   └── [examinationId]/ # 個別診察画面（医師/衛生士）
│   │
```

### 3.2 各画面の詳細

---

#### A. 訪問計画スケジュール（カレンダー）

**パス:** `/dental/schedule`

**概要:** 月間カレンダー形式で訪問計画を管理

**UI要素:**
- 月間カレンダー表示（月切り替え可能）
- 施設フィルター（ドロップダウン）
- 「+ 新規作成」ボタン
- 予定がある日に施設名ラベル表示
- 予定クリック → 訪問計画詳細へ遷移

**機能:**
- 訪問計画の一覧表示
- 新規訪問計画の作成
- 施設でのフィルタリング

---

#### B. 訪問計画詳細（並び替え・診察前設定）

**パス:** `/dental/schedule/[visitPlanId]`

**概要:** 当日の訪問対象者の選択・診察順序の設定

**UI要素（2カラムレイアウト）:**

| 左カラム | 右カラム |
|----------|----------|
| **施設登録患者リスト** | **本日の訪問・診察リスト** |
| - 建物×フロアでグループ化 | - 診察対象者カード |
| - ドラッグ&ドロップで追加 | - 担当医/DH選択 |
| - チェックマークで選択状態表示 | - 診察(医)/指導(衛)ボタン |

**ヘッダー:**
- 日付表示
- 施設名
- 「訪問全体を終了する」ボタン

**機能:**
- 患者リストのドラッグ&ドロップによる順序変更
- 建物×フロアでの自動ソート
- 担当者（医師/衛生士）の割り当て
- 診察画面への遷移

---

#### C. 診療画面 ✅ モック実装済

**パス:** `/dental/consultation/[examinationId]`

**概要:** 診察中の記録入力画面（医師モード/衛生士モード）

**UI要素:**

**ヘッダー部:** ✅
- 患者情報（建物・部屋番号・氏名）
- ログインユーザー表示
- 担当・モード表示
- 「訪問補助」チェックボックス

**タイマーセクション:** ✅
- DR（医師）タイマー + 開始/終了ボタン
- DH（歯科衛生士）タイマー + 開始/終了ボタン
- 経過時間表示
- 20分経過アラート（視覚的フィードバック）

**バイタル測定:** ✅
- 処置前（血圧、SpO2、体温、その他）
- 処置後（タップで入力）
- 「パス」可能

**本日の実施項目（チェックボックス）:** ✅
- 在宅管（管理）
- 訪衛指（指導）
- 口腔機能（検査/訓練）
- 施設系（会議等）
- その他の処置（自由入力 + 追加ボタン）

**実施記録・所見:** ✅（4項目形式で実装）
- 訪問時の様子
- 口腔内所見
- 処置
- 次回予定

**実施項目の選択（加算）:** ✅
- ON/OFFトグルスイッチ
- インフォマーク（PDF説明リンク）
- 該当区分選択（ラジオボタン / 択一式）
- 「自動判定」ボタン
- 加点件数表示
- 必要な提供文書ボタン
- 合計点数計算・表示

**機能:**
- ✅ 診療時間の自動計測
- 🚧 開始・終了時刻の手動変更（変更履歴管理）
- ✅ 20分経過アラート
- 🚧 過去履歴参照（モーダル表示）
- 🚧 音声メモのAI要約

### 4.2 届出資格（クリニック設定）

| 資格名 | 説明 | 型 |
|--------|------|-----|
| 歯訪診 | 歯科訪問診療料の注13に規定する基準 | boolean |
| 歯援診 | 在宅療養支援歯科診療所 | enum (1/2/なし) |
| 在歯管 | - | boolean |
| 口管強 | 歯科口腔リハビリテーション料2の注6に規定する施設基準 | boolean |
| 在宅歯科医療情報連携加算 | **保留** | boolean |
| ベースアップ加算 | **保留** | boolean |
| DX加算 | **保留** | boolean |

## 5. 機能要件

### A. スケジュール・患者管理（訪問前〜訪問時）

| 機能 | 詳細 |
|------|------|
| 施設・フロア管理 | 施設名、フロア（棟・階数）でのグルーピング表示 |
| 訪問順序調整 | 患者リストのドラッグ&ドロップによる回診順序の変更 |
| 建物×フロア自動ソート | 効率的な回診順序の自動提案 |

### B. 診療実施モード（訪問時）

| 機能 | 詳細 |
|------|------|
| タイマー機能 | 診療開始・終了時刻の記録、20分経過アラート |
| 時刻手動変更 | 開始/終了時刻の手動変更（変更履歴管理付き） |
| 担当者モード切替 | 「歯科医師」「歯科衛生士」ごとの入力モード |
| 前回履歴参照 | 過去の診察記録（SOAP記録や特記事項）のモーダル表示 |
| 音声メモAI要約 | 音声入力からのAI要約生成 |

### C. 記録・入力（SOAP・点数）

| 機能 | 詳細 |
|------|------|
| SOAP入力 | 訪問時の様子、口腔内所見、処置、次回予定 |
| 実施項目選択 | 算定項目ごとのチェックボックス |
| 該当区分選択 | 項目ONの時、該当区分を択一選択 |
| 自動判定 | 諸条件に基づく該当区分の自動判定 |
| 点数計算 | 項目×該当区分の組み合わせで点数を自動計算 |
| 提供文書作成 | テンプレートからのデータ流し込み、PDF保存 |

### D. サマリー（診療後）

| 機能 | 詳細 |
|------|------|
| 日次報告 | 本日の実施患者数、算定点数概算の一覧表示 |

### E. マスタ管理

| 機能 | 詳細 |
|------|------|
| 施設マスタ | 訪問先施設の登録・編集・削除 |
| 利用者マスタ | 施設別の利用者管理 |
| スタッフマスタ | 医師・衛生士の管理 |
| クリニック設定 | 届出資格の設定 |

## 6. UIデザイン方針

| 方針 | 詳細 |
|------|------|
| タッチ操作最適化 | タブレットでの操作を想定し、ボタンサイズや入力エリアを大きめに確保 |
| 視認性 | 高齢者施設での利用も考慮し、コントラスト高めの配色 |
| ステータス色分け | 未＝グレー、中＝青、済＝緑 |
| 情報階層 | 常に「今誰を診ているか」「時間はどれくらい経過したか」がわかるヘッダー配置 |
