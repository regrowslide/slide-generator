## 1. 実施項目の選択 ✅

### 概要

診療画面にて、「算定項目一覧」（別紙添付資料）にある各種項目の実施有無を選択する。

### UI仕様

- ON/OFFトグルボタンで各項目の実施有無を選択
- 各ボタン内に以下を表示:
  - 項目名（略称）
  - 項目解説リンク（ℹ️マーク → 説明用PDFを開く）
  - ONの時のみ追加表示:
    - **該当区分**: 択一式（該当区分の選択が不要な項目もある）
    - **自動判定ボタン**: 諸条件に基づき該当区分を自動判定
    - **加点件数**: 項目×該当区分の組み合わせで点数を自動計算
    - **必要な提供文書一覧**: ボタン形式で表示、クリックで文書作成ページへ遷移

### 自動判定ロジック

以下の条件に基づき該当区分を自動判定する。初期値は自動判定結果を使用し、手動変更可能。手動変更後に「自動判定」ボタンで戻せる。

| 判定キー | 条件 | 判定結果 |
|----------|------|----------|
| `clinicKoukankyou` | `clinic.qualifications.koukukan === true` | 口腔管の届出あり |
| `clinicQualification` | `shiensin1 === true` → 歯援診1、`shiensin2 === true` → 歯援診2、いずれもなし → なし | 歯援診の区分 |
| `clinicDxSetting` | `clinic.qualifications.dx === true` | DX加算の適用判定 |
| `hasOralHypofunction` | `patient.hasOralHypofunction === true` | 口腔機能低下症の有無 |
| `consecutiveVisits` | 同一月内の訪問回数に基づく | 連続訪問判定 |

### 算定項目マスタ（PROCEDURE_ITEMS_MASTER）

全42+項目を7カテゴリに分類:

| カテゴリ | 項目数 | 主な項目 |
|----------|--------|----------|
| 歯訪系 | 5 | 初診(291点)、再診(56点)、歯援診1(340点)、歯援診2(230点)、在補診(60点) |
| 在宅管系 | 5 | 歯在管(260-340点)、在口管(60-130点)、小訪(205-325点) |
| 訪衛指系 | 3 | 訪衛指(360-520点)、訪衛指管加(120点)、小多訪衛(305-380点) |
| リハ系 | 4 | 口リハ1(305-395点)、口リハ2(320点)、口リハ管(100点) |
| 検査系 | 5 | 口腔機能評価(100点)、舌圧(140点)、咀嚼(140点)、咬合(10点)、嚥下内視鏡(670点) |
| 歯周系 | 6 | SP/SRP/P処(各種点数) |
| その他 | 5 | 情報連携(120点)、DX加算(10点)、施設総管(700-990点) |

各項目には以下のプロパティがある:
- `id`, `name`, `category`
- `points`（固定点数）または `categories`（区分ごとの点数）
- `autoJudge`（自動判定キー）
- `requiredClinicQualification`（必要な届出資格ID）
- `requiredDocuments`（必要な提供文書テンプレートID）

## 2. 提供文書の作成 ✅（基本実装）/ 🚧（PDF生成・S3保存）

### 概要

実施項目のうち必要な提供文書がある場合、該当の「提供文書ボタン」から専用ページへ遷移して文書の作成・保存を実施。

### 実装済み

- 文書作成画面の基本UI
- テンプレートからのデータ流し込み表示
- 自動項目（クリニック名、施設名、患者名等）の自動挿入
- 手動項目の入力フォーム

### 未実装（`unimplemented.md` 参照）

- PDF生成（`@react-pdf/renderer` or `jspdf`）
- AWS S3への保存
- バージョン管理
- ダウンロード機能

## 3. 届出・施設基準の設定 ✅

### 概要

クリニック設定画面にて、届出・施設基準をチェックボックスで管理する。

### UI仕様

- 青背景ヘッダー「届出・施設基準」
- 全8項目をチェックボックスで表示
- 「その他」のみテキスト入力フィールド付き

### 資格一覧（CLINIC_QUALIFICATIONS）

| ID | 名称 | 型 |
|----|------|----|
| `shiensin1` | 歯援診1 | boolean |
| `shiensin2` | 歯援診2 | boolean |
| `zahoshin` | 在補診 | boolean |
| `koukukan` | 口腔管 | boolean |
| `johorenkei` | 在宅歯科医療情報連携加算 | boolean |
| `dx` | 在宅医療DX情報活用加算 | boolean |
| `baseup` | ベースアップ加算 | boolean |
| `other` | その他 | boolean + `otherText` (string) |

### 算定項目との連携

- 資格のON/OFFが算定項目の自動判定に影響
- `requiredClinicQualification` が設定された算定項目は、対応する資格がONでないと選択不可

## 4. 患者情報入力・アセスメント ✅

### 概要

患者編集画面にて、基本情報・基礎疾患・アセスメントを入力する。2カラムレイアウト。

### 左パネル: 患者情報入力

#### 基本情報
- 患者ID（表示のみ）
- 氏名
- 生年月日（→ 年齢を自動算出）

#### 基礎疾患（PATIENT_DISEASES: 14項目）

チェックボックス形式。青背景セクションヘッダー。

| ID | 名称 |
|----|------|
| `dementia` | 認知症 |
| `hypertension` | 高血圧症 |
| `cerebrovascular` | 脳血管障害 |
| `mentalDisorder` | 精神疾患 |
| `parkinsons` | パーキンソン病 |
| `heartFailure` | 心不全 |
| `terminalCancer` | 末期がん |
| `senility` | 老衰 |
| `femurFracture` | 大腿骨頸部骨折 |
| `spinalStenosis` | 脊柱管狭窄症 |
| `als` | 筋萎縮性側索硬化症（ALS） |
| `cerebellarDegeneration` | 腎臓小脳変性症 |
| `multipleSclerosis` | 多発性硬化症 |
| `disuseSyndrome` | 廃用症候群 |

#### 認薬
- 薬品名 × 3（テキスト入力）

#### チェック項目
- 診療情報等連携共有料（情共1）
- 総合医療管理加算（総医）

### 右パネル: 患者状態・アセスメント

ラジオボタンによる択一選択形式。青背景セクションヘッダーで各カテゴリを区分。

#### 基本情報
- 身長 (cm)、体重 (kg)、BMI（自動計算）

#### 誤嚥性肺炎の既往
- 無し / あり（+ 日付入力）
- 繰り返しあり（+ 日付入力）

#### 身体・口腔状況

| 項目 | 選択肢 |
|------|--------|
| 座位保持 (JZ3-1) | 良好 / やや不良 / 不良 |
| 口腔清掃の状況 | 自主的に行える / 声がけのみ必要 / 一部お手伝いが必要 / 全介助が必要 |
| お口の中の水分保持 | 無理なく可能 / 困難 / 不可能(むせる) / 飲んでしまう / 口から出てしまう |
| うがい | 可能 / 困難 / 不可能（むせる） / 飲んでしまう / 口から出てしまう |

#### リスク・摂取

| 項目 | 選択肢 |
|------|--------|
| 低栄養リスク | なし / 少しあり / リスク高め |
| むせ | なし / 液体で時々 / 頻繁にある |
| 経口摂取の有無 | 全て経口摂取 / 一部経口摂取 / 経口摂取なし |

#### 栄養・水分

| 項目 | 選択肢 |
|------|--------|
| 人工栄養法 | 無し / 胃瘻 / 経鼻 / 抹消静脈 / 中心静脈 / その他 |
| 水分 | トロミなし / トロミあり / 経口摂取禁止 |

#### 食事・服薬

| 項目 | 選択肢 |
|------|--------|
| 食形態（主食 JZ12-1～3） | 常食 / 軟飯 / ペースト食 |
| 食形態（おかず JZ12-4～5） | 常食 / 一口大・軟菜 / 刻み食 / ミキサー食 |
| 飲み込み | 問題なし / 時々むせることがある / 頻繁にむせてしまう |
| 薬の服用 | 問題なし / 苦手 / 上手く飲み込めない |

## 5. 診療画面のタイマー ✅

### 概要

診療中にDR（歯科医師）/DH（歯科衛生士）それぞれの診療時間を計測する。

### 仕様

- 開始/停止ボタン（DR/DH別）
- 経過時間のリアルタイム表示（mm:ss形式）
- **20分アラート**: 20分経過時に視覚的な警告表示
- 開始時刻・終了時刻の記録

---

## 6. バイタル測定 ✅

### 概要

処置前・処置後のバイタルサイン（血圧・体温・SpO2）を記録する。

### 項目

| 項目 | 単位 | 処置前 | 処置後 |
|------|------|--------|--------|
| 血圧（収縮期/拡張期） | mmHg | ✅ | ✅ |
| 体温 | ℃ | ✅ | ✅ |
| SpO2 | % | ✅ | ✅ |

---

## 7. 訪問計画スケジュール ✅

### 概要

月間カレンダー形式で訪問予定を管理する。

### 仕様

- 月表示カレンダー（月切り替え可能）
- 施設フィルター（ドロップダウン）
- 予定がある日に施設名ラベル表示
- 予定クリック → 訪問計画詳細へ遷移
- 「+ 新規作成」ボタン（施設・日付を選択して新規訪問計画を作成）

---

## 8. 訪問計画詳細 ✅

### 概要

当日の訪問対象者の選択・診察順序の設定を行う。

### 2カラムレイアウト

| 左カラム: 施設登録患者リスト | 右カラム: 本日の訪問・診察リスト |
|------|------|
| 建物×フロアでグループ化 | 診察対象者カード |
| クリック/+ボタンで右カラムに追加 | 担当医(DR)/担当衛(DH)選択 |
| チェックマークで選択済み表示 | 診察(医)/指導(衛)ボタンで診療画面へ |

### ヘッダー

- 日付、施設名
- 「訪問全体を終了する」ボタン → 日次報告画面へ

---

## 9. 個別入力 ✅

### 概要

スケジュール外で直接患者を選択して診療を開始する。

### フロー

1. **施設選択**: ドロップダウンから施設を選択
2. **患者選択**: 選択した施設の患者一覧から選択
3. **スタッフ選択**: 担当医(DR)・担当衛(DH)を選択
4. **診察開始**: adhocなvisitPlan + examinationを自動作成し、診療画面へ遷移

## 11. 算定対象台帳 ✅

### 概要

全患者×主要算定項目のマトリクステーブルで、算定状況を一覧管理する。

### UI仕様

- **行**: 患者
- **列**: 主要算定項目（初診、歯在管、訪衛指、口リハ1 等）
- **セル色分け**:
  - 🔴 赤: 要算定 / 算定期間超過
  - 🟢 緑: 算定済 / 期間内
  - ⚪ グレー: 未算定
- **★アラートインジケーター**: 注意が必要な項目にマーク
- 凡例表示

---

## 12. 最終提示画面 ✅

### 概要

診療完了後のサマリーを確認する画面。

### 表示項目

- 診療情報（患者名、施設名、日付、担当者、診療時間）
- 実施記録（訪問時の様子 / 口腔内所見 / 処置 / 次回予定）
- 算定項目一覧（選択した項目、区分、点数）
- 合計点数

## 13. マスタ管理 ✅

### 施設マスタ

- CRUD操作（追加、編集、削除）
- 並び替え（▲▼ボタン）
- 施設種別の設定

### 利用者マスタ

- 施設別の利用者管理
- CRUD操作
- 建物・フロア・部屋番号の設定

### スタッフマスタ

- 2カラムレイアウト（歯科医師 / 歯科衛生士）
- CRUD操作
- 並び替え（▲▼ボタン）

### テンプレート登録

- 提供文書テンプレートの参照表示
- テンプレート名、正式名称、関連算定項目、自動/手動項目数

## 14. ダッシュボード ✅

### 概要

アプリのトップ画面。主要機能へのナビゲーションと本日のサマリーを提供。

### UI要素

- 統計サマリー（本日の訪問予定数、診察完了数、日付）
- 4つのナビゲーションカード:
  - Schedule / Visits（訪問計画スケジュール）
  - Patient Management（患者管理）
  - Individual Input（個別入力）
  - Master Data Management（マスタデータ管理）

## 15. サイドバーナビゲーション ✅

```
🏠 トップ
─── 予定・訪問 ───
📅 訪問計画スケジュール
─── 患者 ───
🔍 患者管理
✏️ 個別入力
─── マスタデータ管理 ───
🏥 クリニック設定
🏢 施設マスタ
👥 利用者マスタ
👨‍⚕️ スタッフマスタ
📋 テンプレート登録
─── レポート・参照 ───

📒 算定対象台帳
```
