# 提供文書機能 構築ルール

## 1. レイアウト原則

### 実物の紙帳票に忠実なHTML再現
- 各文書は、実際に使われている紙の帳票（PDFテンプレート）のレイアウトを **そのまま** HTML `<table>` で再現する
- 左パネル（入力欄）＋右パネル（プレビュー）の分割UIは **使わない**
- 1つの統合されたHTMLフォームとして描画し、帳票そのものが直接編集可能な画面となる

### テーブル構成
- 帳票全体を1つの連続した `<table>` で構成する（ヘッダーからフッターまで）
- 左列にラベル、右列に入力・表示項目を配置
- セルの結合（`rowSpan`, `colSpan`）を使い、紙帳票の罫線構造を忠実に再現する
- 罫線は `border-collapse: collapse` + `border` で表現

## 2. インタラクション原則

### 全チェックボックス・ラジオボタンは操作可能
- 画面上のすべてのチェックボックス（☑/☐）およびラジオボタンは、クリックで切り替え可能にする
- 表示専用（読み取り専用）のチェックボックスは作らない
- `Chk` コンポーネントには `onToggle` プロパティを必ず渡し、クリックイベントを処理する

### テキスト入力
- テキスト欄は `<input>` または `<textarea>` で直接編集可能にする
- プルダウン選択肢がある項目は `<select>` または選択式UIを使用する
- 数値入力は `type="number"` で適切な `min`/`max`/`step` を設定する

## 3. 自動引用（Auto-Quote）ルール

### 初期状態は空
- 文書を開いた時点では、すべての項目は **空の初期値** とする
- 患者マスタや診察データからの自動入力は行わない

### 「自動引用する」ボタン
- 各文書の上部に「自動引用する」ボタンを配置する
- ボタンを押した時のみ、患者マスタ・診察データ・クリニック情報から値を引用・反映する
- ボタンは `print:hidden` とし、印刷時には表示しない

### 自動入力項目のハイライト
- 自動引用によって値が入った項目は、**黄色背景**（`bg-yellow-100`）でハイライト表示する
- ハイライトにより、どの項目が自動入力されたかを視覚的に区別できるようにする
- ハイライトされた項目も手動で上書き編集が可能

### 引用データソース
| データソース | 引用される項目の例 |
|---|---|
| 患者マスタ | 氏名、生年月日、年齢、性別、疾患名、服薬、肺炎既往、食事形態、アセスメント |
| 診察記録 | 訪問時様子、口腔内所見、処置内容、次回予定、Dr/DH時間 |
| クリニック情報 | 医療機関名、所在地、電話番号 |
| 口腔機能精密検査 | 検査値、該当チェック（口腔機能管理計画書で使用） |

## 4. コンポーネント設計

### 共通コンポーネント
- **`Chk`**: チェックボックス表示。`checked`, `label`, `onToggle` を受け取る
- **`AutoQuoteBar`**: 自動引用ボタン＋ハイライト凡例。全文書で共通使用
- **`hlClass`**: ハイライト判定ヘルパー。`highlighted` Set とフィールド名から CSS クラスを返す

### 状態管理パターン
```typescript
// 各文書のデータ: useState で空の初期値
const [data, setData] = useState<DataType>(getDefaultData)

// 引用データ: useMemo で documentData から生成
const quoteData = useMemo(() => buildQuoteData(documentData), [documentData])

// ハイライト管理: プレビューコンポーネント内部で管理
const [highlighted, setHighlighted] = useState<Set<string>>(new Set())
```

### 自動引用の実装フロー
1. ユーザーが「自動引用する」ボタンをクリック
2. `autoQuoteData`（親から渡された引用データ）を現在の `data` にマージ
3. マージされたフィールドのキーを `highlighted` Set に追加
4. State 更新によりUIが再描画され、該当項目が黄色ハイライトされる

## 5. 印刷対応

- 操作ボタン類（自動引用ボタン、保存ボタン等）は `print:hidden` で印刷時非表示
- ハイライト背景は印刷時に影響しない程度の薄い色を使用
- テーブルの罫線・レイアウトは印刷時もそのまま出力される想定
